## 🔴 KNOWN-001：Hook DLL 注入易被杀软拦截

**影响模块**: hook_core  
**影响程度**: 用户体验  
**优先级**: P0（高）  
**计划修复版本**: MVP+1  

### 问题描述

Hook DLL 注入技术（使用 `SetWindowsHookEx` 或 `CreateRemoteThread`）容易被杀毒软件误报为恶意行为，导致：
- 杀毒软件弹窗警告
- DLL 注入失败
- 应用功能受限

### 影响范围

- 所有使用 Hook 技术的功能
- 用户首次运行时最容易遇到

### 临时解决方案

1. **添加数字签名（EV 证书）** - 已完成
2. **加入杀毒软件白名单** - 在文档中提供指导
3. **使用更隐蔽的注入技术** - 计划 MVP+1 实现

### 修复计划（MVP+1）

- 研究更隐蔽的 DLL 注入技术（如 APC 注入、Thread Hijacking）
- 实现多种注入技术的自动选择
- 添加注入失败后的降级方案

### 验证方法

- [ ] 在 Windows Defender 环境下测试，无警告
- [ ] 在第三方杀毒软件（360、腾讯电脑管家）环境下测试，无警告
- [ ] DLL 注入成功率 ≥ 95%

## 🟠 KNOWN-002：注册表 Hive 对比算法复杂

**影响模块**: vreg  
**影响程度**: 功能完整性  
**优先级**: P1（中）  
**计划修复版本**: MVP+1  

### 问题描述

注册表 Hive 对比算法（用于对比虚拟注册表与真实注册表的差异）过于复杂，导致：
- 代码难以维护
- 性能开销较大
- 边界情况处理不完善

### 影响范围

- 注册表虚拟化功能
- 多版本应用隔离

### 临时解决方案

- 当前算法可以正常工作，只是复杂度较高
- 在文档中说明已知限制

### 修复计划（MVP+1）

- 重构注册表 Hive 对比算法
- 使用更高效的数据结构（如前缀树）
- 优化性能，降低开销
- 完善边界情况处理

### 验证方法

- [ ] 代码复杂度降低（Cyclomatic Complexity < 15）
- [ ] 性能提升（对比速度提升 ≥ 50%）
- [ ] 边界情况测试通过率 100%

## 🟡 KNOWN-003：应用直接调用 Syscall 绕过 Hook

**影响模块**: hook_core  
**影响程度**: 兼容性  
**优先级**: P2（低）  
**计划修复版本**: MVP+2  

### 问题描述

部分应用直接调用 Windows Syscall（绕过 Win32 API），导致 Hook 失效：
- Hook 基于 Win32 API 拦截
- 直接 Syscall 调用无法拦截
- 兼容性受限

### 影响范围

- 使用直接 Syscall 调用的应用
- 部分游戏和反作弊软件

### 临时解决方案

- 在文档中说明已知限制
- 提供兼容应用列表

### 修复计划（MVP+2）

- 研究 Syscall 级别 Hook 技术（如 SSDT Hook、Syscall Hook）
- 实现 Win32 API Hook + Syscall Hook 双层拦截
- 测试兼容性和稳定性

### 验证方法

- [ ] Syscall 调用拦截成功率 ≥ 90%
- [ ] 兼容性测试通过率 ≥ 95%
- [ ] 性能开销 < 10%
