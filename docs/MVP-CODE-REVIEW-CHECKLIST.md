# MVP 代码评审检查表

> 本文档是 AI ThinApp Portable Launchpad Platform 项目 MVP Week 10 的代码评审检查表。
> 用于确保代码质量、性能和安全性达到 MVP 标准。
> 日期：2026-05-23 | 评审人：Qt/C++ 性能优化工程师

---

## 使用说明

### 如何使用本检查表

1. **逐条检查**：对每个模块，逐条检查以下检查项。
2. **评分**：每项按 0-10 分评分（0 = 未实现，10 = 完美）。
3. **记录问题**：对每项扣分的地方，记录具体问题和改进建议。
4. **计算总分**：每个维度总分 = 各项得分之和 ÷ 项数 × 10。
5. **判断是否通过**：总分 ≥ 80 分（代码质量）、≥ 80 分（性能）、≥ 80 分（安全）则通过。

### 评分标准

| 分数 | 说明 |
|------|------|
| 0-3 | 未实现或存在严重问题（崩溃、数据丢失、安全漏洞） |
| 4-6 | 基本实现，但存在明显缺陷（性能差、边界条件未处理） |
| 7-8 | 实现良好，但存在改进空间（代码可读性、边缘情况） |
| 9-10 | 实现优秀，符合最佳实践（性能优化、错误处理完善） |
