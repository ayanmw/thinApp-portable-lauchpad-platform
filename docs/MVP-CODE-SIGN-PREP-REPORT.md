## 2. EV 证书申请流程摘要

### 2.1 证书选择建议

**推荐**: **Sectigo EV Code Signing Certificate**

| 证书提供商 | 价格（1 年） | 优点 | 缺点 |
|-----------|--------------|------|------|
| **Sectigo** | $299 | 性价比高，支持中国公司申请 | 品牌知名度略低于 DigiCert |
| **DigiCert** | $399 | 品牌知名度高，技术支持好 | 价格稍贵 |
| **GlobalSign** | $299 | 亚洲市场支持好，中文技术支持 | - |

### 2.2 申请流程概览

**总预计时间**: 10-21 工作日（约 2-4 周）

| 阶段 | 预计时间 | 关键活动 |
|------|----------|----------|
| 准备材料 | 1-2 工作日 | 收集营业执照、电话账单、申请人身份证 |
| 提交申请 | 1 小时 | 在线填写申请表、上传验证材料、支付费用 |
| 公司身份验证 | 1-3 工作日 | CA 机构验证公司真实性（电话回访） |
| 电话验证 | 1-2 工作日 | CA 机构拨打公司电话验证申请人身份 |
| 证书审批 | 1-2 工作日 | CA 机构最终审批 |
| USB Token 制作 | 3-5 工作日 | 将证书写入 USB Token |
| Token 邮寄 | 3-7 工作日 | 快递邮寄 USB Token |

## 3. 代码签名脚本清单

### 3.1 脚本列表

| 脚本名称 | 路径 | 用途 | 使用方法 |
|---------|------|------|----------|
| `sign_ev.ps1` | `tools/ci/sign_ev.ps1` | 使用 EV 证书签署可执行文件 | `.\sign_ev.ps1 -FilePath "app.exe"` |
| `sign_timestamp.ps1` | `tools/ci/sign_timestamp.ps1` | 为已签名的文件添加时间戳 | `.\sign_timestamp.ps1 -FilePath "app.exe"` |
| `sign_batch.ps1` | `tools/ci/sign_batch.ps1` | 批量签署多个可执行文件 | `.\sign_batch.ps1 -Directory ".\bin" -Recurse` |
| `verify_signature.ps1` | `tools/ci/verify_signature.ps1` | 验证签名是否有效 | `.\verify_signature.ps1 -FilePath "app.exe" -Detailed` |

### 3.2 脚本依赖

**必需软件**:
1. **Windows SDK** - 包含 `signtool.exe`
   - 下载地址: https://developer.microsoft.com/en-us/windows/downloads/windows-sdk/
   - 安装时勾选"Windows 10 SDK"或"Windows 11 SDK"

**可选硬件**:
1. **EV 代码签名 USB Token** - 用于存储 EV 证书
   - 随 EV 证书一起邮寄
   - 需要安装驱动程序（通常自动安装）

### 3.3 脚本使用流程

**典型签名流程**:

```powershell
# 1. 插入 USB Token

# 2. 签署单个文件
.\tools\ci\sign_ev.ps1 -FilePath ".\bin\AI-ThinApp.exe"

# 3. 添加时间戳（推荐）
.\tools\ci\sign_timestamp.ps1 -FilePath ".\bin\AI-ThinApp.exe"

# 4. 验证签名
.\tools\ci\verify_signature.ps1 -FilePath ".\bin\AI-ThinApp.exe" -Detailed

# 5. 批量签署（可选）
.\tools\ci\sign_batch.ps1 -Directory ".\bin" -Recurse
```

**MVP 测试阶段（使用自签名证书）**:

```powershell
# 1. 创建自签名证书
.\tools\ci\create_self_signed_cert.ps1 -CertName "AI ThinApp Test" -ValidityDays 365 -ExportPath ".\test_cert.pfx"

# 2. 使用自签名证书签名（需要指定证书指纹）
$signtool = "C:\Program Files (x86)\Windows Kits\10\bin\10.0.22621.0\x64\signtool.exe"
& $signtool sign /fd SHA256 /sha1 <证书指纹> /tr http://timestamp.sectigo.com ".\bin\AI-ThinApp.exe"
```

## 4. 已知限制

### 4.1 硬件限制

| 限制 | 说明 | 解决方案 |
|------|------|----------|
| **需要 USB Token** | EV 代码签名证书必须存储在 FIPS 140-2 Level 2 或更高标准的 USB Token 中 | 购买 EV 证书时会随附 USB Token |
| **Token 可能丢失** | USB Token 是物理设备，可能丢失或损坏 | 备份证书（从 Token 导出为 PFX 文件） |
| **Token PIN 码** | Token 有 PIN 码保护，连续输错可能锁定 | 妥善保管 PIN 码，记录密码提示问题 |

### 4.2 费用限制

| 限制 | 说明 | 解决方案 |
|------|------|----------|
| **EV 证书费用高** | EV 证书年费 $299-399，对 MVP 项目可能负担较重 | 考虑先使用 OV 证书或自签名证书 |
| **时间戳服务器依赖** | 签名时依赖外部时间戳服务器，服务器故障时无法签名 | 配置多个备用时间戳服务器（已在脚本中实现） |

### 4.3 流程限制

| 限制 | 说明 | 解决方案 |
|------|------|----------|
| **申请周期长** | EV 证书申请需要 2-4 周，可能延误 MVP 发布 | 立即开始申请流程；MVP 测试阶段使用自签名证书 |
| **公司验证严格** | EV 证书需要严格的公司身份验证，可能不通过 | 确保公司信息真实有效，电话账单地址与营业执照一致 |
| **不支持个人申请** | EV 代码签名证书只能由公司申请，个人无法申请 | 如果有公司，使用公司名义申请；如果没有，考虑 OV 证书 |

### 4.4 技术限制

| 限制 | 说明 | 解决方案 |
|------|------|----------|
| **SmartScreen 声誉累积** | 即使是 EV 签名，SmartScreen 也可能需要一段时间才能建立声誉 | 使用 EV 证书可以立即获得基本声誉，但仍需用户下载量累积 |
| **时间戳服务器兼容性** | 不同 CA 的时间戳服务器可能不兼容 | 使用通用的时间戳服务器（如 Sectigo、DigiCert） |
| **签名后无法修改** | 签名后任何对文件的修改都会导致签名失效 | 确保在最终版本上签名，使用 CI/CD 自动化签名流程 |

## 6. 风险与缓解措施

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| EV 证书申请被拒绝 | MVP 无法获得有效签名，用户看到"未知发布者"警告 | 低 | 确保公司信息真实有效；如果被拒绝，使用 OV 证书或自签名证书 |
| USB Token 丢失 | 无法签名，需要重新申请证书 | 中 | 备份证书（导出为 PFX 文件）；妥善保管 Token |
| 时间戳服务器故障 | 无法添加时间戳，证书过期后签名失效 | 低 | 配置多个备用时间戳服务器（已在脚本中实现） |
| 编译延迟导致签名时间不足 | MVP 发布延误 | 中 | 提前准备签名脚本；确保编译通过后 30 分钟内可完成签名 |
| SmartScreen 仍然显示警告 | 用户不信任，下载量少 | 中 | 使用 EV 证书可以立即获得基本声誉；通过用户下载量累积声誉 |
