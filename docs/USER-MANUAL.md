## 第 1 章：快速开始

本章带你从零开始，完成 AI ThinApp Portable Launchpad 的下载、安装和首次使用。

### 1.1 产品简介

AI ThinApp Portable Launchpad 是一款个人软件便携化平台。简单来说，它能做到：

- **每个软件独立隔离**：软件的所有配置、缓存、数据文件都保存在软件自己的目录下，不会在 C 盘或其他系统目录产生垃圾文件
- **复制即恢复**：重装系统或换电脑后，复制软件目录即可完整恢复使用状态，无需重新安装和配置
- **干净卸载**：删除软件目录即彻底清理，不留任何残留

**适合哪些人**：

- 喜欢绿色软件、便携化软件的用户
- 系统洁癖用户（不想 C 盘被各种软件污染）
- 需要在多台电脑间同步软件环境的用户
- 频繁重装系统的用户（希望快速恢复工作环境）

### 1.2 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 22H2 或更高版本 / Windows 11 23H2 或更高版本 |
| 处理器 | x64（64 位）处理器 |
| 内存 | 建议 4 GB 以上 |
| 磁盘空间 | 至少 500 MB 可用空间（不含应用本身） |
| 权限 | 需要管理员权限（沙箱引擎注入时需要） |

> **注意**：不支持 Windows 7、Windows 8/8.1 及更早版本。

### 1.3 下载

前往 [GitHub Releases 页面](https://github.com/anmingwei/ai-thinApp-portable-launchpad-platform/releases) 下载最新版本。

**下载步骤**：

1. 打开浏览器，访问 [GitHub Releases](https://github.com/anmingwei/ai-thinApp-portable-launchpad-platform/releases)
2. 找到最新版本（如 v0.1.0）
3. 下载 `ai-thinapp-v0.1.0-setup.exe`（安装程序）
4. （可选）下载 `SHA256SUMS` 文件，用于验证安装包完整性

**验证安装包完整性**（推荐，确保安全）：

1. 打开命令提示符（Win + R，输入 `cmd`，回车）
2. 进入安装包所在目录（如 `cd Downloads`）
3. 输入命令：`certutil -hashfile ai-thinapp-v0.1.0-setup.exe SHA256`
4. 对比输出的哈希值与 `SHA256SUMS` 文件中的值是否一致

### 1.4 安装

**安装步骤**：

1. 双击 `ai-thinapp-v0.1.0-setup.exe`
2. 如果弹出"用户账户控制"提示，点击"是"
3. 阅读许可协议（GPL v3），勾选"我同意"，点击"下一步"
4. 选择安装路径（默认 `C:\Program Files\AI ThinApp\`），点击"下一步"
5. 点击"安装"，等待安装完成
6. 勾选"启动 AI ThinApp Launchpad"，点击"完成"

### 1.5 添加杀软白名单

> **重要**：由于本产品使用沙箱引擎技术（类似病毒行为的 API Hook），可能会被杀毒软件误报。请先添加白名单，否则功能可能无法正常使用。

**Windows Defender**：

1. 打开"Windows 安全中心" → "病毒和威胁防护"
2. 点击"管理设置" → "排除项"
3. 点击"添加排除项" → "文件夹"
4. 选择 `C:\Program Files\AI ThinApp\`（或你的安装目录）

**火绒安全**：

1. 打开火绒 → "安全设置" → "信任区"
2. 点击"添加信任文件"
3. 选择 `ai-thinapp.exe` 和 `hook_engine.dll`（在安装目录下）

**其他杀软**：

在杀软设置中找到"白名单"或"信任列表"，将 `ai-thinapp.exe` 和 `hook_engine.dll` 添加进去。

### 2.1 添加应用

有三种方式添加应用到 Launchpad：

#### 方式一：从软件商店安装（推荐新手）

软件商店是社区维护的应用集合，包含已验证的便携化应用。

**操作步骤**：

1. 点击左侧导航栏的"软件商店"
2. 浏览热门应用或在搜索框中输入应用名称
3. 点击应用卡片上的"安装"按钮
4. 等待下载和安装完成（进度条显示下载进度）
5. 安装完成后，应用自动出现在"我的应用"列表中

> [截图占位：软件商店界面]
> 应截取：软件商店页面，展示应用列表、搜索框、分类标签

**软件商店功能一览**：

| 功能 | 说明 |
|------|------|
| 热门应用 | 展示社区下载量最高的应用 |
| 分类浏览 | 按办公、开发、设计、游戏等分类查看 |
| 搜索 | 在搜索框中输入应用名称 |
| 评分和评论 | 查看其他用户的评价 |
| 版本管理 | 回滚到旧版本（如有更新） |

#### 方式二：添加本地应用

如果你已经安装了某个应用（如 Notepad++、Chrome），可以将其添加到 Launchpad 管理，使其运行在沙箱中。

**操作步骤**：

1. 点击"我的应用"右上角的"+ 添加应用"按钮
2. 在弹出的对话框中，点击"浏览本地应用"
3. 选择应用的可执行文件（如 `C:\Program Files\Notepad++\notepad++.exe`）
4. 填写应用名称（默认使用文件名，可修改）
5. （可选）添加分类标签（如"开发"、"办公"）
6. 点击"确认"

> **说明**：添加本地应用后，Launchpad 会在应用目录下创建沙箱环境（`VFS` 文件夹）。所有配置和数据将保存在沙箱中，不污染系统。

#### 方式三：导入 .vapp 包

如果你有 .vapp 格式的应用包（如朋友分享的便携化应用），可以导入安装。

**操作步骤**：

1. 点击"+ 添加应用"
2. 选择"从 .vapp 文件安装"
3. 选择 .vapp 文件
4. 选择安装路径
5. 点击"安装"

### 2.2 启动应用

**启动步骤**：

1. 在"我的应用"列表中，找到要启动的应用
2. 点击应用卡片上的"启动"按钮
3. 应用将以沙箱模式启动（默认）

**启动模式说明**：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **沙箱模式**（默认） | 应用运行在隔离环境中，文件/注册表写入被重定向到应用目录内的沙箱文件夹，不污染系统 | 日常使用（推荐） |
| **真实模式** | 应用在真实系统环境中运行，写入真实的系统文件和注册表 | 需要访问系统资源的特殊场景（如打印机驱动、系统工具） |

**切换启动模式**：

1. 右键点击应用卡片
2. 选择"启动设置"
3. 选择"沙箱模式"或"真实模式"
4. 点击"启动"

> [截图占位：启动设置对话框]
> 应截取：启动设置对话框，展示沙箱模式/真实模式选择、启动参数输入框、工作目录选择

**多实例管理**：

部分应用（如浏览器）支持同时运行多个独立实例，每个实例拥有独立的沙箱环境，互不影响：

1. 右键点击应用卡片
2. 选择"启动新实例"
3. 新实例将在独立沙箱中运行

**启动参数配置**：

1. 右键点击应用卡片 → "启动设置"
2. 在"启动参数"栏中输入参数（如 `--no-sandbox`）
3. 在"工作目录"栏中选择启动时的工作目录
4. 点击"保存"

### 2.3 停止应用

**停止步骤**：

1. 在"我的应用"列表中，找到正在运行的应用（状态显示为"运行中" 🟢）
2. 点击应用卡片上的"停止"按钮
3. 应用将正常关闭

> **注意**：如果应用中有未保存的数据，停止前会弹出保存提示。请先保存再停止。

### 2.4 删除应用

> **重要**：删除应用会删除应用目录及所有沙箱数据，操作不可恢复。

**删除步骤**：

1. 右键点击应用卡片
2. 选择"删除应用"
3. 在确认对话框中，选择删除范围：

| 选项 | 说明 |
|------|------|
| 仅删除应用引用 | 保留应用文件和沙箱数据，仅从 Launchpad 列表中移除 |
| 删除应用和沙箱数据 | 删除应用目录及所有沙箱数据（彻底删除） |
| 仅删除沙箱数据 | 保留应用文件，仅清除沙箱中的配置和数据（重置应用） |

4. 点击"确认"

> [截图占位：删除应用确认对话框]
> 应截取：删除应用确认对话框，展示三个删除选项

## 第 3 章：沙箱管理

本章介绍如何查看、导出和清除应用的沙箱数据。

### 3.1 什么是沙箱？

沙箱是每个应用的独立隔离运行环境。简单来说：

- **正常情况**：应用运行时，会把配置文件、缓存、临时文件写入系统盘（C 盘）、ProgramData 等位置
- **沙箱模式**：应用运行时，所有写入都被重定向到应用目录下的 `VFS` 文件夹中，不污染系统

**好处**：

- 系统盘不被污染，始终保持干净
- 应用配置和数据与应用文件在一起，方便备份
- 卸载应用时，删除目录即可彻底清理
- 重装系统后，复制目录即可恢复

### 3.2 查看沙箱文件

每个应用都有一个独立的沙箱目录，存储在该应用目录下的 `VFS` 文件夹中。

**查看步骤**：

1. 在"我的应用"列表中，点击应用卡片进入详情页
2. 点击"打开目录"按钮，打开资源管理器查看应用目录
3. 进入 `VFS` 文件夹，可以看到以下内容：

| 文件夹/文件 | 对应真实路径 | 说明 |
|-------------|-------------|------|
| `C\Windows\` | `C:\Windows\` | 应用对 Windows 目录的写入 |
| `C\Program Files\` | `C:\Program Files\` | 应用对 Program Files 的写入 |
| `AppData\Roaming\` | `%APPDATA%` | 应用的配置和数据 |
| `AppData\Local\` | `%LOCALAPPDATA%` | 应用的缓存和临时数据 |
| `ProgramData\` | `%PROGRAMDATA%` | 系统数据 |
| `Temp\` | `%TEMP%` | 临时文件 |
| `registry.hive` | 虚拟注册表 | 应用对注册表的写入 |

**沙箱内容浏览器**（Launchpad 内置）：

1. 在应用详情页，点击"沙箱内容浏览器"
2. 浏览器以树形结构展示沙箱文件，方便浏览和搜索
3. 双击文本文件可以直接查看内容
4. 搜索框可以按文件名搜索

> [截图占位：沙箱内容浏览器界面]
> 应截取：沙箱内容浏览器窗口，展示 VFS 文件夹的树形结构

### 3.3 导出沙箱

你可以将应用的沙箱数据导出为 .vapp 包，方便备份或分享给朋友。

**导出步骤**：

1. 在应用详情页，点击"导出沙箱"按钮
2. 选择导出格式：
   - **.vapp 包**（推荐）：压缩格式，体积小，便于分享
   - **目录**：未压缩的文件夹，方便查看内容
3. 选择保存路径
4. 点击"导出"

> [截图占位：导出沙箱对话框]
> 应截取：导出沙箱对话框，展示格式选择和路径选择

**导出后的用途**：

- **备份**：保存 .vapp 包，重装系统后导入恢复
- **分享**：发送 .vapp 包给朋友，导入后即可使用相同的应用环境
- **迁移**：将应用迁移到其他电脑

### 3.4 清除沙箱

清除沙箱数据后，应用将恢复到初始安装状态（所有配置、数据、缓存全部丢失）。

**清除步骤**：

1. 在应用详情页，点击"清除沙箱"按钮
2. 确认对话框中显示将被清除的数据列表和大小
3. 如有需要，先点击"导出备份"保存一份
4. 点击"确认清除"

**适用场景**：

- 应用出现异常，需要重置到初始状态
- 想清理应用的缓存和临时文件，释放空间
- 准备重新安装应用

> **警告**：清除沙箱是不可逆操作，请谨慎操作。建议在清除前导出备份。

## 第 5 章：高级功能

本章介绍应用捕获、.vapp 包导入/导出和兼容性规则等高级功能。

### 5.1 应用捕获（AppCapture）

应用捕获工具可以将已安装的普通应用"捕获"为便携化的 .vapp 包，无需重新安装即可在其他电脑使用。

**适用场景**：

- 将本地安装的应用便携化
- 为朋友打包便携版应用
- 备份应用环境

**捕获流程**（3 步向导）：

#### 第 1 步：开始捕获

1. 点击"+ 添加应用" → "捕获应用"
2. 输入应用名称（如"MyApp"）
3. 点击"开始捕获"
4. 工具会自动拍摄系统快照（记录当前文件系统和注册表状态）

## 第 6 章：高级技巧

本章介绍注册表重定向和进程继承等高级技巧。

### 6.1 注册表重定向机制

**原理**：

沙箱引擎通过 Hook 技术拦截应用对注册表的读写操作，将其重定向到应用目录下的 `VFS\registry.hive` 虚拟注册表文件中。

**注册表重定向规则**：

| 注册表路径 | 重定向目标 | 说明 |
|------------|------------|------|
| `HKCU\Software\<应用名>` | `VFS\registry.hive` | 应用用户配置 |
| `HKLM\Software\<应用名>` | `VFS\registry.hive` | 应用系统配置 |
| `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` | 不重定向 | 启动项，需要写入真实注册表 |

**查看虚拟注册表**：

1. 在应用详情页，点击"打开目录"
2. 进入 `VFS` 文件夹
3. 找到 `registry.hive` 文件（可以使用注册表编辑器加载查看）

**手动编辑虚拟注册表**：

1. 打开注册表编辑器（`regedit.exe`）
2. 点击"文件" → "加载配置单元"
3. 选择 `VFS\registry.hive` 文件
4. 输入配置单元名称（如"MyApp"）
5. 在 `HKEY_LOCAL_MACHINE\MyApp` 下查看和编辑注册表项
6. 编辑完成后，点击"文件" → "卸载配置单元"

> **注意**：编辑虚拟注册表时，请确保应用未运行，否则可能导致数据损坏。

### 6.2 进程继承机制

**原理**：

当 Launchpad 启动应用时，沙箱引擎会注入 Hook DLL 到应用进程。应用启动的子进程（如浏览器打开新标签页）会继承父进程的 Hook 状态，自动运行在沙箱中。

**进程继承规则**：

| 场景 | 行为 |
|------|------|
| 应用启动子进程 | 子进程自动继承 Hook 状态，运行在沙箱中 |
| 子进程启动孙进程 | 继续继承 Hook 状态 |
| 外部进程启动应用 | 不会继承 Hook 状态，需要由 Launchpad 启动才能进入沙箱 |

**查看进程树**：

1. 启动应用后，打开任务管理器
2. 在"详细信息"选项卡中，查看应用进程和子进程
3. 所有子进程都应加载 `hook_engine.dll`

**进程继承的高级配置**：

在应用详情页的"高级设置"中，可以配置进程继承行为：

| 选项 | 说明 |
|------|------|
| 启用进程继承 | 默认开启，子进程自动继承 Hook 状态 |
| 排除子进程 | 指定某些子进程不继承 Hook（如外部工具） |
| 注入延迟 | 设置 Hook DLL 注入的延迟时间（毫秒），用于兼容某些特殊应用 |

### 6.3 沙箱数据手动编辑

**场景**：需要手动修改沙箱中的数据文件或配置文件。

**步骤**：

1. 在应用详情页，点击"打开目录"
2. 进入 `VFS` 文件夹
3. 根据需要的文件路径，找到对应的虚拟路径文件夹：
   - 应用对 `C:\Windows\` 的写入 → `VFS\C\Windows\`
   - 应用对 `%APPDATA%` 的写入 → `VFS\AppData\Roaming\`
   - 应用对 `%LOCALAPPDATA%` 的写入 → `VFS\AppData\Local\`
4. 使用文本编辑器或专用工具编辑文件
5. 保存后，应用下次启动会读取修改后的文件

> **注意**：手动编辑沙箱文件时，请确保应用未运行，否则修改可能不生效或被覆盖。

### 6.4 多应用共享沙箱（高级）

**场景**：多个应用需要共享某些配置或数据（如浏览器和 Flash 插件）。

**步骤**：

1. 在 Launchpad 设置中，启用"高级模式"
2. 在应用详情页的"高级设置"中，配置"共享沙箱路径"
3. 指定多个应用使用同一个沙箱目录
4. 这样，多个应用可以共享配置和数据

> **警告**：多应用共享沙箱是高级功能，可能导致应用间冲突。建议仅在确实需要时使用，并确保应用之间兼容。

## 第 7 章：卸载 ThinApp 平台

本章介绍如何完全卸载 AI ThinApp Portable Launchpad 平台。

### 7.1 标准卸载（推荐）

**卸载步骤**：

1. 打开"设置" → "应用" → "应用和功能"
2. 在应用列表中找到"AI ThinApp Portable Launchpad"
3. 点击"卸载"
4. 按照卸载向导提示完成卸载

**卸载向导选项**：

| 选项 | 说明 |
|------|------|
| 保留应用沙箱数据 | 卸载 Launchpad，但保留所有应用的沙箱数据（方便以后重新安装后恢复） |
| 删除所有数据 | 完全卸载，删除 Launchpad 和所有应用的沙箱数据 |

> **建议**：如果计划以后重新安装 Launchpad，选择"保留应用沙箱数据"。这样可以保留应用配置和数据，重新安装后无需重新配置。

### 7.2 手动清理（卸载不干净时）

如果标准卸载后仍有残留文件，可以手动清理：

**清理步骤**：

1. 删除安装目录（默认 `C:\Program Files\AI ThinApp\`）
2. 删除用户配置目录：
   - `%APPDATA%\AI ThinApp\`
   - `%LOCALAPPDATA%\AI ThinApp\`
3. 删除应用数据目录（如果选择了"保留应用沙箱数据"）
4. 删除快捷方式（桌面、开始菜单）
5. 清理注册表（可选，高级用户）：
   - 打开 `regedit.exe`
   - 删除 `HKCU\Software\AI ThinApp`
   - 删除 `HKLM\Software\AI ThinApp`（需要管理员权限）

### 7.3 卸载后恢复应用

如果卸载时选择了"保留应用沙箱数据"，重新安装 Launchpad 后可以恢复应用：

**恢复步骤**：

1. 重新安装 Launchpad
2. 打开 Launchpad
3. 点击"+ 添加应用" → "浏览本地应用"
4. 选择应用目录中的可执行文件
5. Launchpad 会自动识别沙箱数据并恢复应用配置

> **提示**：如果应用目录存储在 C 盘以外（如 D 盘），重装系统时只需要格式化 C 盘，D 盘数据不受影响。重新安装 Launchpad 后即可直接使用。

## 第 9 章：故障排除

本章帮助你解决常见的使用问题。

### 9.1 Hook 注入失败

**症状**：启动应用时提示"Hook 注入失败"或应用无法进入沙箱模式。

**排查步骤**：

| 步骤 | 操作 |
|------|------|
| 1 | 确认以管理员权限运行 Launchpad（右键 → "以管理员身份运行"） |
| 2 | 确认 AI ThinApp 相关文件已在杀软白名单中（见第 1 章 1.5 节） |
| 3 | 确认目标应用不是以管理员权限运行的（权限冲突会导致注入失败） |
| 4 | 检查 Hook 引擎日志：设置 → Hook 设置 → 启用详细日志，查看 `%LOCALAPPDATA%\AI ThinApp\Logs\hook_engine.log` |
| 5 | 尝试重启 Launchpad 后重新启动应用 |
| 6 | 如果仍无法解决，切换到"真实模式"运行 |

### 9.2 应用启动失败

**症状**：点击"启动"后，应用无响应、闪退或卡住。

**排查步骤**：

| 步骤 | 操作 |
|------|------|
| 1 | 确认应用本身是正常的（尝试在资源管理器中直接运行应用） |
| 2 | 切换启动模式（沙箱模式 ↔ 真实模式），看是否仅在沙箱模式下失败 |
| 3 | 清除沙箱数据后重试（可能是沙箱数据损坏） |
| 4 | 检查应用是否使用了反作弊系统或其他安全模块（如游戏反作弊、杀毒软件），这些可能与沙箱引擎冲突 |
| 5 | 查看 Hook 引擎日志，查找错误信息 |
| 6 | 到 [GitHub Issues](https://github.com/anmingwei/ai-thinApp-portable-launchpad-platform/issues) 报告问题，附上日志和复现步骤 |

### 9.3 沙箱文件丢失

**症状**：应用配置或数据丢失，检查沙箱目录发现文件不见了。

**排查步骤**：

| 步骤 | 操作 |
|------|------|
| 1 | 确认应用目录没有被误删除或移动 |
| 2 | 检查 Windows 回收站是否有被删除的文件 |
| 3 | 检查是否有备份的 .vapp 包 |
| 4 | 如果有备份，使用"从 .vapp 文件安装"恢复 |
| 5 | 如果沙箱被手动清除，数据无法恢复，请重新安装应用 |

**预防建议**：

- 重要应用定期导出 .vapp 备份
- 不要随意删除应用目录下的 `VFS` 文件夹
- 移动应用目录时，请通过 Launchpad 中的"移动应用"功能操作，而非直接拖拽文件夹

### 9.4 杀软误报

**症状**：杀毒软件提示 AI ThinApp 相关文件或被沙箱化的应用有威胁。

**原因**：本产品使用 API Hook 技术（与病毒行为类似），可能被杀毒软件误报。

**解决步骤**：

| 步骤 | 操作 |
|------|------|
| 1 | 将 `C:\Program Files\AI ThinApp\` 添加到杀软白名单 |
| 2 | 将 `ai-thinapp.exe` 和 `hook_engine.dll` 单独添加到信任列表 |
| 3 | 如果是第三方应用被误报（因为注入了 Hook DLL），将该应用也加入白名单 |
| 4 | 如果不确定是否安全，可以查看项目源码（开源项目，代码完全透明） |
| 5 | 报告误报到 [GitHub Issues](https://github.com/anmingwei/ai-thinApp-portable-launchpad-platform/issues)，我们会联系杀软厂商处理 |

### 9.5 性能下降

**症状**：开启沙箱模式后，应用运行明显变慢或卡顿。

**排查步骤**：

| 步骤 | 操作 |
|------|------|
| 1 | 确认未开启 Hook 引擎"详细日志"（详细日志会显著影响性能） |
| 2 | 如果不需要文件重定向，可以在应用设置中关闭"文件写入重定向" |
| 3 | 如果不需要注册表重定向，可以关闭"注册表写入重定向" |
| 4 | 如果性能仍然不可接受，切换为"真实模式" |
| 5 | 关注新版本发布，我们持续优化沙箱引擎性能 |

> **目标性能**：Hook 开启后文件操作延迟增幅 < 5%，应用启动时间增幅 < 10%。

### 9.6 重装系统后恢复

**症状**：重装系统后，应用配置和数据丢失。

**恢复步骤**：

1. 确保重装前备份了应用目录（如 `D:\ThinAppApps\`）
2. 安装 AI ThinApp Portable Launchpad
3. 将备份的应用目录复制回原位置
4. 打开 Launchpad，所有应用自动识别
5. 启动应用，确认配置和数据都在

> **提示**：如果应用目录存储在 C 盘以外（如 D 盘），重装系统时只需要格式化 C 盘，D 盘数据不受影响。安装 Launchpad 后即可直接使用。
