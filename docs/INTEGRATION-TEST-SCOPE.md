# 集成测试范围定义 (Integration Test Scope)

**项目:** AI ThinApp Portable Launchpad Platform
**POC 阶段:** Week 4 (最终集成测试)
**文档版本:** V1.0
**更新日期:** 2026-05-23

---

## 1. 测试范围

| 测试类型 | 描述 | 优先级 |
| :--- | :--- | :--- |
| 模块间接口测试 | Hook 引擎 ↔ 应用捕获工具（快照数据格式是否兼容） | P0 |
| 模块间接口测试 | 应用捕获工具 ↔ Launchpad UI（.vapp 包格式是否能被 Launchpad 识别） | P0 |
| 端到端流程测试 | 完整流程：安装应用 → 捕获 → 生成 .vapp → 安装 .vapp → 启动应用 → 验证便携性 | P0 |
| 异常场景测试 | 捕获失败、Hook 注入失败、Launchpad 崩溃等异常场景 | P1 |
| 性能测试 | Hook 引擎对应用性能的影响（启动时间、文件 IO 速度） | P2 |

---

## 2. 测试环境

| 环境 | 说明 | 用途 |
| :--- | :--- | :--- |
| 本机（Win10 22H2） | 主要测试环境 | 所有集成测试 |
| 虚拟机 A（Win10 22H2） | 干净系统 | 验证跨机器便携性 |
| 虚拟机 B（Win11 23H2） | 不同 Windows 版本 | 验证系统版本兼容性 |

---

## 3. 测试数据准备

| 数据类型 | 来源 | 用途 |
| :--- | :--- | :--- |
| 基准应用安装包 | `tests\fixtures\`（由 week4-test-env-prep Agent 下载） | 端到端流程测试 |
| 已捕获的 .vapp 包 | 应用捕获工具生成 | Launchpad 安装测试 |
| 测试脚本 | `tools\ci\run-integration-tests.ps1` | 自动化集成测试 |

---

## 4. 测试退出标准

1. 所有 P0 优先级测试用例执行完成，通过率 100%
2. P1 优先级测试用例通过率 ≥ 90%
3. 无阻塞级别的缺陷（Blocker）
4. 集成测试报告通过评审
