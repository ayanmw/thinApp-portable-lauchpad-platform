## 3. 测试范围

### 3.1 编译验证

**目标**：验证所有模块可成功编译

**任务**：
1. 打开 Visual Studio Developer Command Prompt (x64)
2. 运行 `tools\build\build_hook_simple.bat`（编译 Hook DLL）
3. 运行 `tools\build\build_launchpad.bat`（编译 Launchpad）
4. 记录编译输出（`docs/MVP-COMPILE-LOG.md`）
5. 如果编译失败，迭代修复 → 编译 → 验证

**验收标准**：
- ✅ Hook DLL 编译成功（`build\output\x64\hook_engine.dll`）
- ✅ Launchpad 编译成功（`build\output\x64\launchpad.exe`）
- ✅ 所有测试程序编译成功（`build\output\x64\test_*.exe`）
- ✅ 编译错误 = 0，编译警告 ≤ 10

### 3.3 集成测试

**目标**：运行集成测试，验证跨模块功能正常

**测试场景**：

| 测试场景 | 描述 | 优先级 |
|----------|------|----------|
| 模块间接口测试 | Hook 引擎 ↔ 应用捕获工具（快照数据格式是否兼容） | P0 |
| 模块间接口测试 | 应用捕获工具 ↔ Launchpad UI（.vapp 包格式是否能被 Launchpad 识别） | P0 |
| 端到端流程测试 | 完整流程：安装应用 → 捕获 → 生成 .vapp → 安装 .vapp → 启动应用 → 验证便携性 | P0 |

| 性能测试 | Hook 引擎对应用性能的影响（启动时间、文件 IO 速度） | P2 |

**验收标准**：
- ✅ 集成测试通过率 ≥ 80%（8/10 个测试用例通过）
- ✅ Hook DLL 注入成功率 = 100%（10/10 个应用注入成功）

**验收标准**：
- ✅ 启动时间 < 2s（测试 100 次，平均值 < 2s）

## 7. 已知限制（POC 阶段未解决的问题）

### 7.1 Hook 引擎

| 限制 | 影响 | 解决方案 |
|------|------|----------|
| Hook 覆盖检测逻辑未完整实现 | 无法自动检测 Hook 被覆盖 | V2 阶段使用 MinHook API 检测 |
| 注册表 Hive 解析未完整实现 | 无法正确加载已保存的 Hive | V2 阶段使用完整的 JSON 解析库 |
| 子进程继承性能监控埋点未完成 | 无法准确测量注入延迟 | 手动编辑 `hook_engine.cpp` 添加埋点 |

### 7.2 打包工具

| 限制 | 影响 | 解决方案 |
|------|------|----------|
| .vapp 包压缩为桩实现 | 压缩/解压性能较低 | V2 阶段集成 zlib 库 |
| 兼容性规则匹配性能较差 | 通配符和正则匹配 > 1ms | V2 阶段使用前缀树优化 |
| 应用商店 API 调用为桩实现 | 无法真实上传/下载应用 | V2 阶段搭建测试 API 服务器 |

### 7.3 Launchpad UI

| 限制 | 影响 | 解决方案 |
|------|------|----------|
| Hook DLL 注入未实际实现 | 应用无法在沙箱中运行 | Week 10 实现 Windows API 注入 |
| 图标缓存未集成到主窗口 | 内存占用可能高于预期 | Week 10 重构主窗口 UI |
| 单元测试未实现 | 代码质量无法保证 | Week 10 编写单元测试 |
