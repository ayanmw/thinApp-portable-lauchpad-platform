## 第 1 章：视频概述

### 1.1 视频目标

本视频旨在帮助新用户快速上手 AI ThinApp Portable Launchpad，完成从安装到第一个便携化应用的完整流程。

### 1.2 目标受众

- 未使用过本产品的用户
- 希望了解"便携化"概念的用户
- 需要在多台电脑间同步软件环境的用户

### 1.3 视频结构

| 章节 | 内容 | 时长（约） |
|------|------|------------|
| 开头 | 产品介绍和价值主张 | 30 秒 |

| **总计** | | **6 分钟** |

### 1.4 拍摄准备

**软件准备**：

- 安装 AI ThinApp Portable Launchpad (v0.1.0)
- 准备一个测试应用（如 Notepad++ x64 便携版）
- 确保杀毒软件已添加白名单

**硬件准备**：

- 屏幕录制软件（如 OBS Studio、Bandicam）
- 麦克风（用于配音）
- 分辨率：1920x1080 (1080p) 或更高

**环境准备**：

- 关闭不必要的应用程序
- 清理桌面和任务栏（避免隐私泄露）
- 使用浅色主题（提高可读性）

### 3.4 沙箱模式和便携化（3:00 - 4:30）

**配音文本（逐字稿）**：

```
第三步，了解沙箱模式和便携化的原理。

AI ThinApp 有两种启动模式：真实模式和沙箱模式。

真实模式，就是让应用直接运行在系统中，
不做任何隔离。
这种模式适合信任的应用，性能也最好。

沙箱模式，则是通过 Hook 引擎，
将应用的文件操作、注册表操作重定向到沙箱目录中，
实现完全隔离。

我们来演示一下沙箱模式。

右键点击应用卡片，选择"启动设置"。
在弹出的对话框中，选择"沙箱模式"，然后点击"保存"。

现在，启动应用。

在沙箱模式下，应用对文件和注册表的所有操作，
都会被重定向到 `Sandbox` 目录中。
这意味着，应用无法修改系统其他位置的文件，
也无法读取其他应用的注册表数据。

如果你不想让某个应用留下任何痕迹，
可以在关闭应用后，删除 `Sandbox` 目录。
这样，应用就恢复到了初始状态，
就像从来没有运行过一样。

这种隔离机制，不仅保护了系统，
也让应用更容易便携化。
因为所有数据都在应用目录下，
你只需要复制整个应用目录，
就能在另一台电脑上完整恢复这个软件的使用状态。
```
