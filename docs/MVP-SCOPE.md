# MVP 范围定义 (MVP Scope)

> 本文档定义 AI ThinApp Portable Launchpad Platform 项目的 MVP（最小可行产品）范围。
> 版本：0.1 | 日期：2026-05-23 | 作者：PM

---

## 1. MVP 目标

### 1.1 核心目标

MVP（Minimum Viable Product）阶段的核心目标是**验证完整的产品闭环**，实现从"应用捕获 → 打包 → 分发 → 运行"的完整流程。

**5 个核心目标**：

1. **实现完整的 Hook 引擎**
   - 文件系统重定向（完整实现 CoW 逻辑、写入重定向）
   - 注册表重定向（完整实现 hive 持久化、权限模拟）
   - 子进程继承（3 层，覆盖 Chrome 多进程等场景）

2. **实现应用捕获工具（AppCapture）和 .vapp 包格式**
   - 快照对比（文件系统 + 注册表）
   - .vapp 包导出（ZIP + JSON 元数据）
   - .vapp 包安装器

3. **实现 Launchpad 完整 UI**
   - 应用管理（列表、启动、停止、删除）
   - 应用商店（浏览、搜索、下载、安装）
   - 托盘常驻 + 通知

4. **解决杀软兼容性问题**
   - 代码签名（OV 或 EV 证书）
   - 杀软白名单申请
   - 误报率 ≤ 10%（VirusTotal 扫描）

5. **发布第一个可用版本（v0.1.0）**
   - GitHub Release（含安装包）
   - 应用商店上线（社区 .vapp 包分发）
   - 用户文档（用户手册、FAQ、视频教程）

### 1.2 成功标准（5 条）

- [ ] **S1**：Hook 引擎稳定运行（24 小时压力测试无崩溃，内存泄漏 < 5MB / 24h）
- [ ] **S2**：3 款基准应用（Notepad++, 7-Zip, Firefox Portable）可正常便携化（文件/注册表重定向正确）
- [ ] **S3**：应用捕获工具可成功捕获 10 款常用应用（捕获成功率 ≥ 90%）
- [ ] **S4**：Launchpad UI 可正常使用（应用管理、启动、商店功能完整）
- [ ] **S5**：杀软误报率 ≤ 10%（VirusTotal 扫描，50+ 引擎）

### 1.3 不包含的功能（明确边界）

见第 3 节"不包含的功能（MVP 阶段）"。

### 2.1 优先级说明

- **P0（Must Have）**：MVP 核心功能，缺失则产品不可用
- **P1（Should Have）**：重要功能，缺失则用户体验严重受损
- **P2（Nice to Have）**：体验优化功能，可延后到 V2
- **P3（Optional）**：可选功能，MVP 阶段可做可不做

## 3. 不包含的功能（MVP 阶段）

明确 MVP 阶段**不做**的功能，避免范围蔓延。

| 功能 | 原因 | 计划阶段 |
|------|------|----------|
| 内核态 Hook（Minifilter 驱动） | 技术复杂度高，WHQL 认证周期长 | V2 阶段 |
| 多用户隔离 | MVP 聚焦单用户场景 | V2 阶段 |
| 企业级管理后台 | 目标用户是 C 端，非企业 | V2 阶段 |
| Linux/macOS 支持 | 目标平台是 Windows | V3 阶段 |
| 全局只读缓存（共用运行时去重） | 性能优化，MVP 不优化性能 | V2 阶段 |
| 进程间通信隔离（命名管道、共享内存） | 复杂度高，POC 已验证基础隔离 | V2 阶段 |
| COM 对象虚拟化 | 复杂度高，仅少数应用需要 | V2 阶段 |
| 云端同步（强制） | 隐私风险，用户可能不接受 | V2 阶段（可选） |
| 开始菜单替换（L6 方案） | 系统稳定性风险高 | V3 阶段（可选） |

## 4. 成功标准（详细）

### 4.1 S1：Hook 引擎稳定性

**验收标准**：
- [ ] 连续运行 24 小时无崩溃
- [ ] 内存泄漏 < 5MB / 24h（任务管理器监控）
- [ ] Hook 安装/卸载 100 次无错误
- [ ] 子进程继承 3 层无失败

**测试方法**：
1. 启动 Hook 引擎
2. 运行 3 款基准应用（Notepad++, 7-Zip, Firefox Portable）
3. 每小时触发一次文件/注册表操作（自动化脚本）
4. 24 小时后检查崩溃日志和内存增长

### 4.2 S2：3 款基准应用可正常便携化

**验收标准**：
- [ ] Notepad++：启动正常，配置保存在应用目录，移动目录后可正常运行
- [ ] 7-Zip：启动正常，右键菜单可用（需注册 DLL），配置文件保存在应用目录
- [ ] Firefox Portable：启动正常，配置文件保存在应用目录，多进程模型正常工作

**测试方法**：
1. 使用 AppCapture 捕获应用
2. 启动应用，执行基本操作（打开文件、修改配置、关闭重启）
3. 检查 `{AppDir}\VFS\` 目录，确认文件/注册表重定向正确
4. 移动应用目录到新位置，重新启动，验证功能正常

### 4.3 S3：应用捕获工具可成功捕获 10 款常用应用

**验收标准**：
- [ ] 捕获成功率 ≥ 90%（10 款应用中至少 9 款成功）
- [ ] 捕获的应用可正常安装和运行
- [ ] 捕获时间 < 5 分钟/应用

**测试应用清单**（待确认）：
1. Notepad++
2. 7-Zip
3. Firefox Portable
4. VSCode
5. VLC Media Player
6. LibreOffice
7. GIMP
8. Inkscape
9. Audacity
10. Thunderbird

**测试方法**：
1. 准备干净系统（虚拟机快照）
2. 运行 AppCapture，拍摄前快照
3. 安装应用（手动或静默）
4. 运行 AppCapture，拍摄后快照
5. 生成 .vapp 包
6. 在另一台机器上安装 .vapp 包，验证应用正常运行

### 4.4 S4：Launchpad UI 可正常使用

**验收标准**：
- [ ] 应用列表加载成功（从配置文件读取）
- [ ] 应用启动成功率 100%（点击"启动"按钮，应用正常运行）
- [ ] 应用停止功能正常（强制结束进程）
- [ ] 应用商店可浏览、搜索、下载、安装 .vapp 包
- [ ] 托盘常驻 + 通知功能正常

**测试方法**：
1. 启动 Launchpad
2. 验证应用列表显示正确（图标、名称、版本）
3. 点击"启动"按钮，验证应用正常启动
4. 点击"停止"按钮，验证应用进程终止
5. 进入应用商店，搜索应用，下载并安装
6. 验证托盘图标显示，点击托盘图标弹出菜单

### 4.5 S5：杀软误报率 ≤ 10%

**验收标准**：
- [ ] VirusTotal 扫描结果：误报率 ≤ 10%（50+ 引擎中 ≤ 5 个报毒）
- [ ] 代码签名证书已安装（OV 或 EV）
- [ ] 主流杀软（Windows Defender、火绒、360）不报毒或仅首次提示

**测试方法**：
1. 上传 Hook DLL 和 Launchpad 可执行文件到 VirusTotal
2. 检查扫描结果，计算误报率
3. 在干净 Windows 环境中安装主流杀软
4. 启动 Hook 引擎和 Launchpad，记录杀软反应

## 5. MVP 与 POC 的关系

### 5.1 POC → MVP 的延续性

| POC 验证项 | MVP 实现目标 | 关系 |
|------------|--------------|------|
| V1：Hook 引擎稳定性 | 修复 POC 限制，实现完整 Hook 引擎 | 延续 + 完善 |
| V2：文件系统重定向 | 修复 CoW 逻辑，实现 VFS 缓存 | 延续 + 优化 |
| V3：注册表重定向 | 优化 hive 格式，实现持久化 | 延续 + 优化 |
| V4：子进程继承 | 修复 CreateProcessInternalW 实现，支持 3 层继承 | 延续 + 完善 |
| V5：杀软兼容性 | 申请代码签名证书，提交白名单申请 | 延续 + 解决 |
| V6：应用验证 | 扩展 to 10 款应用 | 延续 + 扩展 |
| V7：跨位置便携性 | 自动化测试覆盖 | 延续 |
| V8：交互原型验证 | 实现真实 Launchpad UI | 原型 → 产品 |

### 5.2 POC 评审不通过的风险

若 POC 评审未通过（No-Go），MVP 计划可能作废。

**可能的原因**：
- V1 失败：Hook 引擎无法稳定工作
- V2 失败：文件系统重定向不完整
- V5 失败：杀软持续报毒且无法申请白名单

**应对措施**：
- 若技术路线不可行，重新评估方案（如切换到内核驱动）
- 若杀软误报无法解决，考虑开源托管 + 用户自编译
