# 技术风险详述 - R-MVP-01（Hook 引擎性能问题）

## 风险描述

Hook 引擎在拦截文件/注册表/进程操作时，若每次操作都执行路径重定向计算+磁盘 IO，会导致：
- 文件操作延迟增加 10-100 倍（用户体验极差）
- 高并发场景（如编译大型项目）下 CPU 占用过高
- 内存占用持续增长（未释放缓存）

## 影响范围

- **影响用户**：所有使用便携化应用的用户
- **影响功能**：文件操作、注册表操作、进程启动
- **影响指标**：应用启动时间、文件保存时间、系统响应速度

## 概率评估

- **技术难度**：中（重定向逻辑复杂，需大量字符串操作）
- **类似项目参考**：Sandboxie Plus 在重定向大量小文件时性能下降明显
- **概率**：**中**（40-60% 概率在 MVP 阶段暴露性能问题）

## 缓释措施（按优先级排序）

### P0（必须实现，MVP Phase 1 完成）

**措施 1：实现 VFS 缓存（内存 + 磁盘）**
- 内存缓存：LUR 策略，缓存最近访问的 1000 个文件路径映射
- 磁盘缓存：将路径映射表保存到 `\{AppDir}\VFS\path_cache.bin`（二进制格式，快速加载）
- 预期性能提升：文件操作延迟降低 80%

**措施 2：优化路径重定向算法**
- 使用前缀树（Trie）存储 VFS 路径映射，查询复杂度 O(n) → O(k)（k 为前缀长度）
- 避免每次都调用 `wcsstr`（O(n*m)）
- 预期性能提升：路径重定向计算速度提升 5-10 倍

### P1（建议实现，MVP Phase 2 完成）

**措施 3：实现异步 IO（适用于大文件操作）**
- 文件读取：使用 `ReadFile` 异步模式（OVERLAPPED）
- 文件写入：先写内存缓存，定时刷盘（每 5 秒）
- 预期性能提升：大文件（>100 MB）操作延迟降低 50%

### P2（可选实现，MVP Phase 3 完成）

**措施 4：实现 IO 优先级调整**
- 降低 Hook 引擎的 CPU/IO 优先级（使用 `SetPriorityClass`）
- 避免 Hook 引擎影响前台应用性能
- 预期性能提升：前台应用响应速度提升 10-20%

## 应急预案（若缓释措施失效）

1. **降级方案**：提供"性能模式"开关，用户可禁用部分 Hook（仅重定向关键路径）
2. **回滚方案**：若性能问题无法解决，回退到 POC 阶段的基础重定向逻辑（牺牲部分功能换取性能）
3. **用户沟通**：在文档中明确说明性能限制（如：不适合处理超大文件）

## 监控指标

- 文件操作延迟（目标：< 10 ms/操作）
- 注册表操作延迟（目标：< 5 ms/操作）
- Hook 引擎内存占用（目标：< 50 MB/应用）
- CPU 占用（目标：< 5% 单核）

若任意指标连续 3 次测试超标，触发应急预案。

## 测试计划

| 测试场景 | 测试方法 | 验收标准 |
|----------|----------|----------|
| 小文件读写（< 1 MB） | 创建 1000 个小文件，测量延迟 | 平均延迟 < 10 ms/操作 |
| 大文件读写（> 100 MB） | 复制大文件，测量延迟 | 延迟 < 未 Hook 时的 2 倍 |
| 注册表操作 | 读取/写入 1000 次注册表项 | 平均延迟 < 5 ms/操作 |
| 内存占用 | 启动 5 个应用，监控内存增长 | 每个应用 < 50 MB |
| CPU 占用 | idle 状态监控 | < 5% 单核 |
