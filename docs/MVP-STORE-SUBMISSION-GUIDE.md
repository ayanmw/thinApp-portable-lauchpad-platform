# MVP 应用商店上线指南

> 本文档为 **AI ThinApp Portable Launchpad Platform** 的 MVP 应用商店上线提供完整指导。
> 目标：编译通过后 1 小时内可提交应用商店审核。

---

## 1. 应用商店选择

### 1.1 平台对比

| 平台 | 适合场景 | 优势 | 劣势 |
|------|---------|------|------|
| **Microsoft Store** | Windows 桌面应用 | 系统集成深、自动更新、企业用户多 | 审核严格、分成比例 85/15 |
| **Steam** | 游戏/工具类应用 | 用户基数大、社区活跃 | 必须走 Steam Direct、$100 费用 |
| **Itch.io** | 独立开发者 | 免费、审核宽松、灵活定价 | 流量较少、需要自建推广 |
| **独立站 (GitHub Pages)** | 技术用户、开源版 | 完全控制、无分成、直接分发 | 需要自行推广、无内置支付 |

### 1.2 推荐策略

**第一阶段（MVP 上线）：**
- ✅ **Microsoft Store** - 主分发渠道（Windows 应用最权威）
- ✅ **Itch.io** - 辅助分发渠道（快速上线、收集反馈）
- ✅ **独立站** - 开源版本 + 文档托管

**第二阶段（用户增长后）：**
- 考虑 Steam（如果用户反馈好、有游戏化需求）

## 2. 上架要求

### 2.1 Microsoft Store

| 项目 | 要求 |
|------|------|
| **开发者账号** | Microsoft Partner Center 账号 |
| **费用** | 个人账号 $19（一次性）、企业账号 $99（一次性） |
| **审核周期** | 通常 24-48 小时 |
| **必需材料** | EV 代码签名证书、应用包 (.msix/.appx)、隐私政策、应用描述、截图 |
| **重要提示** | 需要 EV 代码签名（硬件令牌或云服务），申请周期 1-2 周 |

**注册链接：** https://partner.microsoft.com/dashboard

### 2.2 Steam

| 项目 | 要求 |
|------|------|
| **开发者账号** | Steam Direct 账号 |
| **费用** | $100 USD / 应用（一次性） |
| **审核周期** | 1-5 天（人工审核） |
| **必需材料** | 应用包、商店页面素材、视频/截图、应用描述 |
| **重要提示** | 需要税务信息、银行账户（收益分成 70/30） |

**注册链接：** https://partner.steamgames.com/

### 2.3 Itch.io

| 项目 | 要求 |
|------|------|
| **开发者账号** | 免费注册 |
| **费用** | 免费（可选捐赠） |
| **审核周期** | 即时上线（无审核） |
| **必需材料** | 应用包、应用描述、截图 |
| **重要提示** | 收益分成自定义（建议 100% 给开发者） |

**注册链接：** https://itch.io/developers

### 2.4 独立站 (GitHub Pages)

| 项目 | 要求 |
|------|------|
| **成本** | 免费（GitHub Pages 免费托管） |
| **域名** | 可选（自定义域名 $10-15/年） |
| **上线周期** | 即时（Git Push 后自动部署） |
| **必需材料** | 静态网站文件、下载链接、文档 |
| **重要提示** | 需配合 GitHub Releases 托管二进制文件 |

## 3. 材料准备

### 3.1 应用图标

| 用途 | 尺寸 | 格式 | 状态 |
|------|------|------|------|
| 应用商店主图标 | 512x512 | PNG (透明背景) | ⏳ 待准备 |
| 小图标 | 256x256 | PNG | ⏳ 待准备 |
| 超大图标 | 1024x1024 | PNG | ⏳ 待准备 |

**设计规范：**
- 透明背景 PNG
- 无圆角（应用商店自动处理）
- 主体清晰、可识别（小尺寸下也能看清）

### 3.2 应用截图

| 用途 | 尺寸 | 格式 | 数量 | 状态 |
|------|------|------|------|------|
| Microsoft Store | 1920x1080 | PNG | 5-8 张 | ⏳ 待准备 |
| Steam | 1920x1080 | PNG | 5-10 张 | ⏳ 待准备 |
| Itch.io | 1920x1080 | PNG | 3-5 张 | ⏳ 待准备 |

**截图内容建议：**
1. 主界面 - Launchpad 启动面板全景
2. 便携应用列表 - 展示已添加的便携软件
3. 沙箱隔离演示 - 虚拟化运行界面
4. 一键启动 - 快速启动动画/截图
5. 设置界面 - 个性化配置

### 3.3 应用描述

| 类型 | 字数限制 | 用途 | 状态 |
|------|---------|------|------|
| 短描述 | ≤100 字 | 搜索结果、列表页 | ⏳ 待准备 |
| 长描述 | ≤2000 字 | 应用详情页 | ⏳ 待准备 |

**差异化卖点（必须突出）：**
- ✅ **便携化**：无需安装，U 盘随身携带
- ✅ **沙箱隔离**：安全虚拟化运行，不污染系统
- ✅ **Launchpad 启动面板**：一键启动所有便携应用
- ✅ **开源免费**：MVP 版本完全免费

### 3.4 法律文档

| 文档 | 必需性 | 用途 | 状态 |
|------|--------|------|------|
| 隐私政策 | ✅ 必需 | 所有平台 | ⏳ 待准备 |
| EULA (最终用户许可协议) | ✅ 必需 | Microsoft Store、Steam | ⏳ 待准备 |
| 开源协议 (LICENSE) | ✅ 必需 | 独立站、GitHub | ✅ 已完成 |

## 4. 提交流程

### 4.1 Microsoft Store 提交流程

```
1. 获取 EV 代码签名证书（提前 1-2 周申请）
   ↓
3. 打包应用 (.msix 格式)
   ↓
4. 使用 EV 证书签名应用包
   ↓
5. 登录 Microsoft Partner Center
   ↓
6. 创建应用、填写元数据
   ↓
7. 上传应用包、提交审核
   ↓
8. 等待审核（24-48 小时）
   ↓
9. 审核通过 → 上线
   审核拒绝 → 修复问题 → 重新提交
```

**自动化脚本：** `tools/deploy/submit_ms_store.ps1`

### 4.2 Steam 提交流程

```
1. 支付 $100 费用（Steam Direct）
   ↓
2. 填写商店页面信息
   ↓
3. 上传应用包、截图、视频
   ↓
4. 设置定价（MVP 建议免费）
   ↓
5. 提交审核
   ↓
6. 等待审核（1-5 天）
   ↓
7. 审核通过 → 上线
```

**自动化脚本：** `tools/deploy/submit_steam.ps1`

### 4.3 Itch.io 提交流程

```
1. 注册 Itch.io 开发者账号（免费）
   ↓
2. 创建新项目
   ↓
3. 填写项目描述、上传截图
   ↓
4. 上传应用包（.zip / .exe）
   ↓
5. 设置定价（建议免费 + 可选捐赠）
   ↓
6. 发布（无审核，即时上线）
```

**自动化脚本：** `tools/deploy/submit_itch.ps1`

### 4.4 独立站部署流程

```
1. 创建 GitHub 仓库
   ↓
2. 构建静态网站（GitHub Pages）
   ↓
3. 配置下载链接（GitHub Releases）
   ↓
4. 部署到 GitHub Pages
   ↓
5. 测试下载、文档链接
```

**自动化脚本：** `tools/deploy/deploy_indie_site.ps1`

## 5. 常见问题

### 5.1 Microsoft Store 常见拒绝原因

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 应用崩溃 | 未充分测试、依赖缺失 | 本地充分测试、使用虚拟机验证 |
| 隐私政策缺失 | 未提供隐私政策链接 | 准备隐私政策文档并上传 |
| 代码签名无效 | 使用普通代码签名（非 EV） | 申请 EV 代码签名证书 |
| 应用描述不符 | 实际功能与描述不一致 | 确保描述准确、不夸大 |
| 包含恶意行为 | 误报（杀毒软件检测） | 提交误报申诉、代码透明化 |

### 5.2 Steam 常见拒绝原因

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 内容质量低 | 应用过于简单、像 demo | 完善功能、确保 MVP 可用性 |
| 缺少必要的素材 | 截图/视频质量差 | 使用高质量截图、添加演示视频 |
| 定价不合理 | 价格与价值不符 | MVP 建议免费或低价 |
| 税务信息不完整 | 未填写税务信息 | 填写 W-8BEN 或 W-9 表格 |

### 5.3 Itch.io 注意事项

- ✅ 无审核，即时上线
- ✅ 支持可选捐赠
- ⚠️ 需要自行推广（无内置流量）
- ⚠️ 建议配合社交媒体、技术社区推广

### 5.4 独立站注意事项

- ✅ 完全控制、无分成
- ✅ 适合开源版本 + 文档
- ⚠️ 需要配合 GitHub Releases 托管二进制文件（单个文件 ≤ 2GB）
- ⚠️ 需要自行配置 CDN 加速（如果下载量大）

## 6. 上线检查清单

### 6.1 编译前检查

- [ ] 代码编译通过（无错误、无警告）
- [ ] 所有功能测试通过（单元测试、集成测试）
- [ ] EV 代码签名证书已获取
- [ ] 应用图标、截图已准备

### 6.2 上线前检查

- [ ] 应用包已签名（EV 代码签名）
- [ ] 隐私政策已准备
- [ ] EULA 已准备
- [ ] 应用描述（短 + 长）已准备
- [ ] 脚本可运行（语法检查通过）

### 6.3 提交后跟踪

- [ ] 记录提交时间、审核编号
- [ ] 每天检查审核状态
- [ ] 准备快速响应审核反馈（如果需要修改）
