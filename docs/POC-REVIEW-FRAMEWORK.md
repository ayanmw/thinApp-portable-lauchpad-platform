# POC 评审框架
本框架用于指导 AI ThinApp Portable Launchpad Platform 项目的 POC 评审工作，确保评审客观、流程清晰、决策有据。

## 1. 评审目标
1. 确认 POC 阶段的所有验证项（V1 ~ V10）已完成或通过
2. 评估技术可行性（Hook 引擎稳定性、文件/注册表重定向正确性）
3. 评估杀软兼容性（误报率、缓解措施）
4. 决策：是否进入 MVP 开发阶段

## 2. 评审标准
| 评审维度 | 权重 | 通过标准 | 评分方法 |
|----------|------|----------|----------|
| 技术可行性 | 30% | Hook 引擎稳定运行，文件/注册表重定向正确率 ≥ 95% | 功能测试通过率 |
| 杀软兼容性 | 20% | 主流杀软误报率 ≤ 10%，且有缓解措施 | VirusTotal 扫描结果 + 杀软测试报告 |
| 用户体验 | 20% | Launchpad 原型可演示，交互流程清晰，可用性测试通过率 ≥ 70% | UX 走查评分 + 可用性测试报告 |
| 便携性 | 15% | 3 款基准应用可跨目录/跨机器运行，无文件泄漏 | 便携性测试通过率 |
| 文档完整性 | 15% | 所有核心文档已输出，且一致性校验通过 | 文档审查清单 |
| **总分** | **100%** | **≥ 70 分通过** | 加权平均 |

## 3. 评审流程
### Day 1：提交评审材料（POC 评审包）
- 截止时间：2026-06-10 18:00
- 提交内容：
  - POC 验证项检查表（`docs/POC-VERIFICATION-CHECKLIST.md`）
  - 技术文档（`SPEC.md`, `ARCHITECTURE.md`, `POC-PLAN.md`）
  - 测试报告（V2、V3、V4、V7、V8 测试报告）
  - 代码审查报告（Hook 引擎代码质量、安全性）
  - 杀软兼容性测试报告
- 提交方式：邮件 + 项目共享目录

### Day 2：技术评审会议（2 小时）
- 时间：2026-06-11 14:00 - 16:00
- 地点：线上（腾讯会议）/ 线下（会议室 A）
- 议程：
  1. Hook 引擎演示（V1、V2、V3、V4）：演示 Hook 安装/卸载、文件/注册表重定向效果
  2. 便携性演示（V7、V8）：演示 3 款基准应用跨目录运行、无文件泄漏
  3. Launchpad 原型演示（V9）：演示核心交互流程（应用启动、沙箱管理）
  4. 杀软兼容性讨论（V6）：讨论误报情况、缓解措施、代码签名计划
- 参会人：项目组全体、技术决策者、测试负责人

### Day 3：决策会议（1 小时）
- 时间：2026-06-12 10:00 - 11:00
- 地点：线上/线下
- 议程：
  1. 评审打分：各评委按照评审标准打分，计算加权总分
  2. 决策：进入 MVP / 延长 POC / 终止项目
  3. 若进入 MVP，制定 MVP 开发计划（10 周）
- 参会人：项目组全体、决策者
