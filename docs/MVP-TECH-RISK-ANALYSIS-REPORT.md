## 2. 逐项分析（高风险项）

### 2.1 R-MVP-01：Hook 引擎性能问题

**风险描述**：Hook 引擎在拦截文件/注册表/进程操作时，若每次操作都执行路径重定向计算+磁盘 IO，会导致性能严重下降。

**影响范围**：
- 影响用户：所有使用便携化应用的用户
- 影响功能：文件操作、注册表操作、进程启动
- 影响指标：应用启动时间、文件保存时间、系统响应速度

**概率评估**：中（40-60% 概率在 MVP 阶段暴露性能问题）

**缓释措施**（详见 [RISK-MVP-01-HOOK-PERFORMANCE.md](./RISK-MVP-01-HOOK-PERFORMANCE.md)）：
- P0：实现 VFS 缓存（内存 + 磁盘）+ 优化路径重定向算法
- P1：实现异步 IO（适用于大文件操作）
- P2：实现 IO 优先级调整

**监控指标**：
- 文件操作延迟（目标：< 10 ms/操作）
- 注册表操作延迟（目标：< 5 ms/操作）
- Hook 引擎内存占用（目标：< 50 MB/应用）

### 2.2 R-MVP-02：应用捕获失败率高

**风险描述**：应用捕获工具（AppCapture）在捕获复杂应用时，可能因应用安装逻辑复杂、动态下载组件、修改系统文件等原因失败。

**影响范围**：
- 影响用户：需要使用非标准应用便携化的用户
- 影响功能：应用捕获、.vapp 包生成
- 影响指标：捕获成功率、.vapp 包质量

**概率评估**：中（40-60% 概率在 MVP 阶段捕获成功率 < 90%）

**缓释措施**（详见 [RISK-MVP-02-APP-CAPTURE-FAILURE.md](./RISK-MVP-02-APP-CAPTURE-FAILURE.md)）：
- P0：优先捕获 10 款常用应用 + 实现快照对比增强
- P1：实现手动调整界面 + 建立兼容性规则库
- P2：实现"录制模式"

**监控指标**：
- 捕获成功率（目标：≥ 90%）
- 捕获时间（目标：< 5 分钟/应用）
- .vapp 包安装成功率（目标：100%）

### 2.3 R-MVP-03：杀软误报率高

**风险描述**：Hook 引擎（DLL 注入 + Inline Hook）是杀毒软件的典型"恶意行为"，会导致主流杀软报毒，用户无法正常运行应用。

**影响范围**：
- 影响用户：所有用户（尤其是企业用户）
- 影响功能：Hook 引擎、Launchpad 主程序
- 影响指标：杀软误报率、用户留存率

**概率评估**：高（70-90% 概率在 MVP 阶段误报率 > 10%）

**缓释措施**（详见 [RISK-MVP-03-ANTIVIRUS-FALSE-POSITIVE.md](./RISK-MVP-03-ANTIVIRUS-FALSE-POSITIVE.md)）：
- P0：申请 OV 代码签名证书 + 提交白名单申请
- P1：申请 EV 代码签名证书 + 开源透明（GitHub 完整源码）
- P2：使用虚拟机隔离（可选功能）

**监控指标**：
- VirusTotal 扫描误报率（目标：≤ 10%）
- 主流杀软（Windows Defender、360、火绒）误报（目标：0）

### 2.4 R-MVP-06：Hook 覆盖检测失效

**风险描述**：Hook 引擎安装的 Hook 可能被其他安全软件或应用程序覆盖，导致 Hook 失效，应用便携化失败。

**影响范围**：
- 影响用户：使用安全软件（如 360、火绒、卡巴斯基）的用户
- 影响功能：Hook 引擎（所有 Hook 点）
- 影响指标：Hook 覆盖率、应用便携化成功率

**概率评估**：高（60-80% 概率在 MVP 阶段遇到 Hook 覆盖问题）

**缓释措施**（详见 [RISK-MVP-06-HOOK-OVERRIDE.md](./RISK-MVP-06-HOOK-OVERRIDE.md)）：
- P0：实现定时检测（每 5 秒）+ 实现自动重装
- P1：实现异常捕获 + 实现 Hook 链管理
- P2：实现内核 Hook（V2 阶段）

**监控指标**：
- Hook 覆盖率（目标：> 95%）
- Hook 重装次数（目标：< 5 次/小时）

### 2.5 R-MVP-07：子进程继承失败

**风险描述**：Hook 引擎需要通过 DLL 注入让子进程也加载 Hook DLL，但可能因 `CreateProcessInternalW` 未 Hook、注入失败等原因导致子进程未继承 Hook。

**影响范围**：
- 影响用户：使用多进程应用的用户（如 Chrome、Firefox、VSCode）
- 影响功能：子进程 Hook 继承
- 影响指标：子进程 Hook 覆盖率、应用便携化成功率

**概率评估**：中（40-60% 概率在 MVP 阶段子进程继承失败）

**缓释措施**（详见 [RISK-MVP-07-CHILD-PROCESS-INHERITANCE.md](./RISK-MVP-07-CHILD-PROCESS-INHERITANCE.md)）：
- P0：完善 `CreateProcessInternalW` Hook + 支持 5 层子进程继承
- P1：多种注入方式 + 注入失败重试
- P2：内核态进程创建拦截（V2 阶段）

**监控指标**：
- 子进程 Hook 覆盖率（目标：> 95%）
- 多级子进程继承成功率（目标：100% 5 层继承）

## 5. 结论

### 5.1 MVP 技术风险是否可控？

**总体结论**：**可控，但需要严格执行缓释措施**。

**理由**：
1. **高风险项均有明确的缓释措施**：所有 5 个高风险项都有 P0 缓释措施（必须实现），且技术可行性高。
2. **缓释措施成本可接受**：总成本 61 人天 + $500-1000/年，在 MVP 预算内。
3. **时间计划可行**：P0 措施在 39 天内完成，团队可以并行工作。
4. **监控方案完善**：每个高风险项都有明确的监控指标和触发阈值，可以及时发现问题。

### 5.2 是否建议继续 MVP 阶段？

**建议**：**继续，但需满足以下条件**：

1. **P0 缓释措施必须全部完成**：若任何一个 P0 措施失败，MVP 成功率将大幅下降。
2. **代码签名证书必须申请成功**：这是解决杀软误报的关键，若失败，MVP 可能无法推广。
3. **团队资源必须到位**：需要 Dev Lead（高级 C++ 工程师）、PM、Sec Lead 等角色，若人员不到位，风险会增加。
