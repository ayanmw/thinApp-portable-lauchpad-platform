# 风险登记册 (Risk Register)

> 本文档记录 AI ThinApp Portable Launchpad Platform 项目的所有已识别风险。
> 更新频率：每周评审一次，新风险随时添加。

---

## 风险矩阵定义

| 影响 \ 概率 | 低 (L) | 中 (M) | 高 (H) |
|------------|--------|--------|--------|
| **致命 (F)** | 中 | 高 | **极高** |
| **高 (H)** | 低 | 中 | 高 |
| **中 (M)** | 低 | 低 | 中 |
| **低 (L)** | 极低 | 低 | 低 |

**风险等级** = 影响 × 概率
- 极高：必须立即处理，项目可能因此终止
- 高：需在下一阶段开始前处理
- 中：需在版本内处理
- 低：可记录观察，不主动处理
- 极低：仅记录

## 附录：风险应对决策树

```
新识别风险
    │
    ▼
影响 × 概率 = 等级？
    │
    ├── 极高 → 立即上报 PM + 决策者，制定应对方案，分配资源
    ├── 高   → 下一阶段开始前处理，指定负责人和截止日期
    ├── 中   → 本版本内处理，记录处理进展
    ├── 低   → 记录观察，不主动处理
    └── 极低 → 仅记录
```
