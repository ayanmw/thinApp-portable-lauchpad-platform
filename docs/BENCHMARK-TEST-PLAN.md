# Week 4 基准应用验证方案 (POC V7)

## 1. 验证目标

本方案旨在验证 AI ThinApp Portable Launchpad Platform 的 Hook 引擎（V7）对基准应用的兼容性与核心功能。具体目标如下：

1.  **Notepad++ 验证**：确认在 Hook 引擎下可正常启动、编辑文本、保存文件，且文件操作被重定向至应用目录。
2.  **7-Zip 验证**：确认在 Hook 引擎下可正常执行压缩与解压操作，且输出文件存储在重定向目录中。
3.  **Firefox Portable 验证**：确认在 Hook 引擎下可正常启动、浏览网页、下载文件，且下载内容存储在 VFS 目录。
4.  **重定向机制验证**：核心目标是验证文件 I/O 和注册表操作是否被正确拦截并重定向到应用专属目录（`VFS` 及 `registry.hive`）。

## 2. 测试环境

为了全面评估兼容性和便携性，设置以下测试环境：

| 环境 | 操作系统 | 说明 | 用途 |
| :--- | :--- | :--- | :--- |
| **本机** | Windows 10 22H2 | 主开发/测试环境 | 功能验证、主要调试 |
| **虚拟机 A** | Windows 10 22H2 | 纯净系统 (未安装待测应用) | 验证应用的便携性 (Portability) |
| **虚拟机 B** | Windows 11 23H2 | 不同 Windows 版本 | 验证系统兼容性 |

**环境准备要求：**
*   本机需已编译 `hook_engine.dll` 及启动器 (`launchpad.exe` 或 `.bat` 脚本)。
*   虚拟机需共享 `D:\MyApps` 目录或通过移动介质挂载，以模拟“便携式”运行场景。

## 3. 测试步骤 (通用流程)

对于每款基准应用，遵循以下标准化测试流程：

### 步骤 1：基线行为记录 (Baseline)
*   **操作**：在不启用 Hook 引擎的正常环境下安装并运行应用。
*   **目的**：记录应用正常的文件关联、注册表写入位置（如 `HKEY_CURRENT_USER\Software\...`）及默认行为，以便与 Hook 后的行为进行对比。

### 步骤 2：Hook 引擎启动应用
*   **操作**：配置环境变量并执行启动脚本。
*   **脚本示例 (`test_launchpad.bat`)**：
    ```batch
    @echo off
    set AI_THINAPP_APP_DIR=D:\MyApps\Notepad++
    set AI_THINAPP_HOOK_DLL=%cd%\hook_engine.dll
    start D:\MyApps\Notepad++\notepad++.exe
    ```
*   **验证点**：应用是否成功启动，有无崩溃或 Hook 报错弹窗。

### 步骤 3：验证文件重定向 (File Redirection)
*   **操作**：通过应用功能打开/保存文件。
*   **验证点**：
    1.  打开文件对话框时，默认路径是否指向 `VFS` 映射的目录。
    2.  编辑并保存文件后，检查物理磁盘上的原始路径是否未被修改，而修改内容仅存在于 `AI_THINAPP_APP_DIR` 下的 `VFS` 对应目录中。

### 步骤 4：验证注册表重定向 (Registry Redirection)
*   **操作**：修改应用设置（如 Notepad++ 的主题、Firefox 的首页）。
*   **验证点**：关闭应用后，检查 `AI_THINAPP_APP_DIR\VFS\registry.hive` 文件是否已更新，且宿主机的注册表中对应项未被修改。

### 步骤 5：验证子进程继承 (Child Process Inheritance)
*   **操作**：触发应用启动子进程（如 Firefox 调用下载器，或 Notepad++ 调用插件进程）。
*   **验证点**：子进程是否自动继承了 Hook 环境，其文件/注册表操作是否同样被重定向。

### 步骤 6：稳定性与压力测试
*   **操作**：长时间运行（1小时），执行高频操作（如 7-Zip 压缩 100 个文件，Notepad++ 快速开关大文件）。
*   **验证点**：监控内存占用（是否有泄漏），CPU 占用是否正常，应用是否崩溃。

## 5. 验收标准

方案成功通过需满足以下条件：

*   [ ] **核心功能**：Notepad++, 7-Zip, Firefox 均能成功启动并执行基本操作 (B-01, B-06, B-09)。
*   [ ] **重定向准确率**：文件操作 100% 重定向至 `VFS`；注册表操作 100% 重定向至 `registry.hive`。
*   [ ] **便携性**：在虚拟机 A (干净系统) 中，应用无需安装即可运行。
*   [ ] **稳定性**：通过 1 小时压力测试，无崩溃，内存增长曲线平稳。
*   [ ] **子进程**：子进程成功继承 Hook 环境。
