## 2. 关键技术方案摘要

### 2.1 技术栈选型

| 技术 | 选型 | 理由 |
|------|------|------|
| **GUI 框架** | **Qt 6** | POC 阶段验证通过，原生性能，LGPL 协议兼容 |
| **Hook 框架** | MinHook | 轻量级，支持 x86/x64，MIT 协议 |
| **构建系统** | CMake | 跨平台，与 Qt 6 集成良好 |
| **编译器** | Visual Studio 2022 | Windows 原生开发工具链 |

**重要决策**：放弃原 `docs/ARCHITECTURE.md` 中的 Electron 方案，改用 **Qt 6**（POC 阶段验证通过）。

### 2.2 核心架构设计

#### 2.2.1 模块划分

```
Launchpad UI 架构
│
├── 主窗口模块 (MainWindow) - 应用列表、详情页、搜索栏
├── 托盘模块 (TrayManager) - 托盘图标、右键菜单、通知
├── 应用管理模块 (AppManager) - 应用列表模型、启动器、持久化
├── 沙箱浏览器模块 (SandboxBrowser) - 目录树、文件列表、可视化
├── 设置模块 (SettingsManager) - 常规设置、Hook 设置、启动项
└── 任务栏集成模块 (TaskbarIntegrator) - 跳转列表、缩略图、进度条
```

#### 2.2.2 信号/槽设计（跨模块通信）

- **AppManager** 发射 `AppLaunched`、`AppStopped`、`AppCrashed` 信号
- **MainWindow** 接收信号更新 UI（应用状态、托盘通知）
- **TrayManager** 接收信号显示 Balloon 通知（启动成功、崩溃、Hook 失败）
- **SandboxBrowser** 与 **AppManager** 交互（文件导出、删除）

#### 2.2.3 线程模型

| 线程 | 职责 | 说明 |
|------|------|------|
| **UI 线程** | 界面渲染、用户交互 | Qt 主事件循环 |
| **Hook 引擎线程** | Hook DLL 注入、API Hook 安装 | 使用 `QtConcurrent` 异步执行 |
| **应用启动线程** | 创建进程、等待启动完成 | 使用 `QProcess::start()` 异步启动 |
| **文件浏览器后台线程** | 遍历沙箱目录、计算文件大小 | 使用 `QtConcurrent::run()` 后台遍历 |
| **应用数据加载线程** | 从配置文件加载应用列表 | 启动时后台加载 |

#### 2.2.4 内存管理

- **QObject 父子关系**：所有 `QObject` 派生类对象必须设置 parent
- **智能指针**：非 QObject 对象使用 `std::unique_ptr`（独占所有权）
- **缓存管理**：应用图标缓存使用 `QCache<QString, QIcon>`

### 2.3 关键技术实现

#### 2.3.1 Hook DLL 注入

```cpp
// 远程线程注入 Hook DLL
bool AppLauncher::InjectHookDll(const QString& appDir, QProcess* process) {
    // 1. 设置环境变量 (AI_THINAPP_APP_DIR)
    // 2. 获取进程句柄 (process->processHandle())
    // 3. 在目标进程中分配内存 (VirtualAllocEx)
    // 4. 写入 DLL 路径到目标进程内存 (WriteProcessMemory)
    // 5. 创建远程线程 (CreateRemoteThread -> LoadLibraryW)
    // 6. 等待远程线程结束 (WaitForSingleObject)
    // 7. 清理 (VirtualFreeEx, CloseHandle)
}
```

#### 2.3.2 沙箱目录树组件

- **技术方案**：自定义模型（`QAbstractItemModel` 子类）
- **理由**：`QFileSystemModel` 无法直接映射虚拟路径（`{AppDir}\VFS\C\Windows\...` → 显示为 `C:\Windows\...`）
- **实现**：`SandboxDirModel` 继承 `QAbstractItemModel`，映射 `{AppDir}\VFS\` 到虚拟路径

#### 2.3.3 任务栏集成

- **跳转列表**：使用 Windows Shell API `ITaskbarList3::SetJumpList()`
- **缩略图工具栏**：使用 Windows Shell API `ITaskbarList3::ThumbBarAddButtons()`
- **进度条**：使用 Windows Shell API `ITaskbarList3::SetProgressValue()`
- **Qt 支持**：Qt 6 未直接封装，需调用 Windows Shell API

## 6. 设计总结

### 6.1 设计亮点

1. **技术栈选型合理**：放弃 Electron 方案，改用 Qt 6（POC 阶段验证通过，性能更优）
2. **架构设计清晰**：模块划分明确，信号/槽设计合理，线程模型安全
3. **用户体验优先**：参考 `docs/UX-RESEARCH.md` 和 `docs/UX-DESIGN.md`，界面符合 Windows 原生规范
4. **性能优化充分**：启动速度、内存、响应速度均有优化策略
5. **风险识别完整**：高风险项已识别并制定缓释措施

### 6.2 待解决问题

1. **任务栏集成复杂度**：需要 Windows Shell API 知识，可能需要学习成本
2. **Hook DLL 注入被杀软拦截**：需要代码签名和杀软白名单申请（见 `docs/MVP-PLAN.md` Week 9）
3. **沙箱文件浏览器性能**：文件数量 > 10000 时可能需要进一步优化

### 6.3 后续计划

1. **Week 7-10 开发**：按照 `docs/MVP-PLAN.md` 和本文档的任务拆解执行
2. **风险监控**：每周评审风险登记册（`docs/RISK-REGISTER.md`），更新风险状态
3. **用户测试**：MVP 完成后进行用户测试，收集反馈并迭代优化
4. **V2 阶段规划**：根据 MVP 反馈和市场需求，规划 V2 阶段功能（内核 Minifilter 驱动、云同步等）
