# 产品规格说明书 (Product Specification)

## 1. 产品概述

**产品名称（暂定）**：AI ThinApp Portable Launchpad Platform  
**英文简称**：APLP  
**GitHub 仓库**：https://github.com/anmingwei/ai-thinApp-portable-lauchpad-platform  

### 1.1 定位
个人软件便携化平台。通过底层 API Hook 实现文件系统/注册表虚拟化，让每个应用的所有运行状态（配置、缓存、数据文件）完全隔离在自身目录内。重装系统或换电脑后，复制应用目录即可完整恢复使用状态。

### 1.2 核心价值
| 痛点 | 解决方案 |
|------|----------|
| 重装系统后软件需要重新安装配置 | 应用目录可移动，复制即恢复 |
| 软件在 C 盘产生大量垃圾文件 | 所有写入重定向到应用目录 |
| 多个软件版本冲突 | 每个软件独立沙箱，互不影响 |
| 软件卸载不干净 | 删除目录即彻底清理 |
| 开始菜单杂乱 | Launchpad 统一管理，可替换开始菜单 |

## 3. 核心功能需求

### 3.1 应用虚拟化引擎（核心）

#### 3.1.1 文件系统虚拟化
- 拦截 `NtCreateFile` / `NtWriteFile` / `NtOpenFile` 等文件系统 API
- 将目标路径重定向到应用沙箱目录内的虚拟路径
  - `C:\Windows\*` → `{AppDir}\VFS\C\Windows\*`
  - `C:\Program Files\*` → `{AppDir}\VFS\C\Program Files\*`
  - `%APPDATA%\*` → `{AppDir}\VFS\AppData\*`
- Copy-on-Write：读取时优先读沙箱，沙箱不存在则透传真实文件
- 支持长路径（>260 字符）

#### 3.1.2 注册表虚拟化
- 拦截 `NtCreateKey` / `NtSetValueKey` / `NtOpenKey` 等注册表 API
- 将 `HKLM\Software\*` 和 `HKCU\Software\*` 写入重定向到虚拟 hive 文件
- 虚拟 hive 格式：基于注册表 hive 格式的自定义实现
- 读取时优先读虚拟 hive，不存在则透传真实注册表

#### 3.1.3 进程隔离
- 拦截 `CreateProcessInternalW` / `ShellExecuteExW` 等进程创建 API
- 子进程自动继承 Hook（DLL 注入）
- 支持 2+ 层子进程继承（覆盖 Chrome 多进程等场景）
- 进程间通信（命名管道、共享内存）可选隔离（V2）

#### 3.1.4 Hook 引擎
- 基于 MinHook 实现用户态 Inline Hook
- 支持 32 位和 64 位进程
- Hook 异常自动恢复（被其他软件覆盖时重新注入）
- 热加载/卸载 Hook 规则

### 3.2 应用打包工具

#### 3.2.1 应用捕获
- 安装前快照 → 安装应用 → 安装后快照 → 差异生成 .vapp 包
- 文件系统差异捕获（文件新增/修改/删除）
- 注册表差异捕获（键/值新增/修改/删除）
- 支持静默安装参数

#### 3.2.2 .vapp 包格式
- 文件头（magic number + 版本 + 元数据偏移）
- 文件数据块（压缩存储）
- 注册表 hive 块
- 元数据 JSON（应用名称、版本、依赖、入口点）

#### 3.2.3 包编辑器（CLI + GUI）
- 查看包内容（文件列表、注册表树）
- 编辑包内容（添加/删除/修改文件或注册表项）
- 合并多个包
- 导出为独立目录或单 .exe

### 3.3 Launchpad 客户端

#### 3.3.1 应用库管理
- 卡片视图 / 列表视图切换
- 应用搜索（名称、标签）
- 应用分类（办公、开发、游戏、工具等）
- 应用状态指示（未安装/已安装/运行中/有更新）

#### 3.3.2 应用启动器
- 一键启动应用（默认沙箱模式）
- 启动模式切换（沙箱 / 真实环境）
- 多实例管理（同时运行多个实例，各自独立沙箱）
- 启动参数配置

#### 3.3.3 软件商店（社区）
- 浏览社区提供的 .vapp 包
- 一键下载安装
- 版本管理（回滚到旧版本）
- 依赖解析（自动安装所需运行时）

#### 3.3.4 设置中心
- 全局 Hook 开关
- 沙箱默认路径配置
- 缓存管理（清理全局缓存）
- 外观设置（主题、布局）

#### 3.3.5 开始菜单替换（V3）
- 可选集成到 Windows 任务栏
- 可选替换 Win 键触发
- 沙箱应用图标角标标识

## 4. 非功能需求

### 4.1 性能
- Hook 开启后文件操作延迟增幅 < 15%（POC 目标），< 5%（MVP 目标）
- Launchpad 启动时间 < 2 秒
- 应用启动时间增幅 < 10%

### 4.2 兼容性
- POC：3 个应用验证通过（Notepad++、7-Zip、Firefox Portable）
- MVP：20+ 应用验证通过
- 体验完善：50+ 应用验证通过
- 支持 Win10 22H2 及以上，x86-64 平台

### 4.3 稳定性
- POC：连续运行 24 小时无崩溃
- MVP：连续运行 7 天无崩溃
- 内存泄漏：< 5MB / 24h

### 4.4 安全性
- 代码开源（GPL v3），接受社区审计
- Hook 行为透明可查（拦截日志）
- 不涉及内核驱动（POC/MVP 阶段），降低蓝屏风险
- 可选云端同步时，用户数据加密存储

### 4.5 易用性
- 新手引导流程：3 步完成首个应用安装和启动
- 默认配置开箱即用，高级选项折叠
- 沙箱概念对用户透明（不需要理解"沙箱"）
- 首次使用引导完成率 ≥ 80%

## 5. POC 验证范围（4 周）

### 5.1 验证项

| ID | 验证项 | 成功标准 | 优先级 |
|----|--------|----------|--------|
| V1 | Hook 引擎稳定性 | 5 个核心 API Hook 安装/卸载无内存泄漏，连续运行 24h 无崩溃 | P0 |
| V2 | 文件系统重定向 | 写入拦截路径的文件出现在沙箱目录；读取时 CoW 正确 | P0 |
| V3 | 注册表重定向 | 注册表写入出现在虚拟 hive；读取时先查虚拟再透传 | P0 |
| V4 | 子进程继承 | 父进程 Hook 传递到子进程（至少 2 层） | P1 |
| V5 | 杀软兼容性 | 在 Windows Defender / 火绒下不误报（或仅首次提示） | P0 |
| V6 | 应用验证 | Notepad++ / 7-Zip / Firefox Portable 可正常运行 | P1 |
| V7 | 跨位置便携性 | 移动应用目录后仍能正常运行 | P1 |
| V8 | 交互原型验证 | 用户能理解"应用隔离"概念，3 步内完成应用启动 | P1 |

### 5.2 Go / No-Go 决策标准
- **Go**：V1 + V2 + V3 + V5 全部通过 → 继续 MVP
- **No-Go**：V1 或 V2 或 V5 任一失败 → 技术路径需调整，暂停投入
- **Conditional Go**：V4/V6/V7/V8 部分失败 → MVP 阶段针对性修复

## 6. MVP 功能范围（10 周）

### 6.1 必须包含
- 稳定的 Hook 引擎（32/64 位，异常恢复）
- 文件系统虚拟化增强（权限模拟、符号链接、长路径）
- 注册表虚拟化增强（hive 持久化、权限模拟）
- 进程模型增强（Chrome 多进程、COM、命名管道）
- 应用捕获工具（快照差异 → .vapp）
- .vapp 包格式 + 包编辑器
- 一键打包（.vapp + 运行时 = 单目录 / 单 .exe）
- Launchpad 主框架（应用库、启动、搜索）
- 软件商店基础（社区 .vapp 分发、下载、安装）
- 设置中心

### 6.2 不包含（推至 V3）
- 开始菜单替换
- 云端同步
- 全局只读缓存（共用运行时去重）
- 企业功能（AD 集成、策略下发）
