## 风险提示

### 高风险（可能影响 POC 评审）

1. **V2/V4 阻塞未解决**（概率 60%，影响 高）
   - **描述**：当前开发环境未安装 Visual Studio 2022，导致 V2（文件系统重定向）、V4（子进程继承）验证项无法编译测试，处于阻塞状态。
   - **影响**：若无法在 2026-05-30 前解决编译环境问题，V2、V4 验证项将无法完成，直接影响 POC 评审中「技术可行性」维度评分（权重 30%），严重时可能导致评审不通过。
   - **缓解措施**：
     - 立即申请安装 VS2022 社区版（免费）
     - 或由 Dev Lead 提供编译服务器权限，临时使用现有编译环境完成验证
     - 若无法及时获得编译环境，评审时提供 V2、V4 代码逻辑审查结果（由 Dev Lead 出具审查意见），作为替代证据，但需在 MVP 阶段补充完整测试

2. **评审材料包未按时提交**（概率 30%，影响 高）
   - **描述**：若待完成任务未能按时完成，评审材料包将无法按时提交。
   - **影响**：评审委员无法及时审阅材料，可能导致评审会议延期。
   - **缓解措施**：
     - 2026-06-09 完成打包，预留 1 天缓冲
     - 若预计延期，提前 3 天通知评审委员，协商调整评审时间安排

### 中风险（可能影响评审结果）

3. **演示视频未录制**（概率 40%，影响 中）
   - **描述**：演示视频是评审时最直观的展示材料，若未录制，评审委员可能无法全面评估 POC 成果。
   - **影响**：评审委员对技术可行性、用户体验的评价可能降低。
   - **缓解措施**：
     - 提前准备录制脚本（`review-package/demos/README.md` 已创建）
     - Week 4 内完成录制（使用 OBS Studio）

4. **测试报告缺失**（概率 50%，影响 中）
   - **描述**：V3、V6、V7、V8 测试报告尚未输出。
   - **影响**：评审材料不完整，可能导致评审委员无法全面评估 POC 成果。
   - **缓解措施**：
     - 优先输出 V3、V6 测试报告（依赖项较少）
     - 若 V7 无法完成，提供基准应用验证计划 + 代码逻辑审查结果作为替代证据
