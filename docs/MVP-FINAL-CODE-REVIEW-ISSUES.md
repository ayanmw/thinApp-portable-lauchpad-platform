## P0 问题（必须修复 - 阻塞 MVP 发布）

### 问题 1：Hook 引擎未完整实现
- **模块**：src/engine/hook/hook_engine.cpp
- **问题描述**：所有 Hook 函数（Hook_NtCreateFile、Hook_NtCreateKey、Hook_CreateProcessW）为桩实现，未实际拦截 API 调用
- **影响**：核心功能不可用，无法进行文件系统和注册表虚拟化
- **修复建议**：实现完整的 Hook 逻辑，使用 MinHook 拦截目标 API
- **预计工作量**：3-5 天

### 问题 2：Hook 完整性检查逻辑错误
- **模块**：src/engine/hook/hook_engine.cpp
- **问题描述**：HookCheckThreadProc 中的 `if (MH_OK != MH_OK)` 永远为 FALSE，无法检测 Hook 覆盖
- **影响**：Hook 覆盖检测失效，安全风险
- **修复建议**：修改为实际检查每个 Hook 的状态
- **预计工作量**：1 天

### 问题 3：VFS 未实现
- **模块**：src/engine/vfs/vfs.cpp
- **问题描述**：所有 VFS 函数（VFS_RedirectPath、VFS_ReadFile、VFS_WriteFile）为桩实现
- **影响**：文件系统虚拟化不可用
- **修复建议**：实现完整的 VFS 逻辑，包括路径重定向、Copy-on-Write
- **预计工作量**：5-7 天

### 问题 4：VReg 未实现
- **模块**：src/engine/vreg/vreg.cpp
- **问题描述**：所有 VReg 函数（VReg_RedirectKeyPath、VReg_CreateKey、VReg_SetValue、VReg_QueryValue）为桩实现
- **影响**：注册表虚拟化不可用
- **修复建议**：实现完整的 VReg 逻辑，包括虚拟 hive 文件读写、Copy-on-Write
- **预计工作量**：5-7 天

### 问题 5：进程继承未实现
- **模块**：src/engine/process/process_inherit.cpp
- **问题描述**：所有进程继承函数（ProcessInherit_Enable、ProcessInherit_InjectDll）为桩实现
- **影响**：子进程无法继承 Hook，多层进程场景失效
- **修复建议**：实现完整的进程继承逻辑，包括 CreateProcess Hook、DLL 注入
- **预计工作量**：5-7 天

### 问题 6：路径遍历攻击防护未实现
- **模块**：src/engine/vfs/vfs.cpp、src/engine/redirect/redirect_engine.cpp
- **问题描述**：未验证文件路径，可能遭受路径遍历攻击（如 `..\..\Windows\system.ini`）
- **影响**：安全漏洞，可能访问敏感文件
- **修复建议**：在所有文件路径操作前，验证路径不包含 `..\` 或使用规范化后的路径
- **预计工作量**：2-3 天

### 问题 7：注册表注入攻击防护未实现
- **模块**：src/engine/vreg/vreg.cpp
- **问题描述**：未验证注册表路径，可能遭受注册表注入攻击（如 `\..\`）
- **影响**：安全漏洞，可能访问敏感注册表项
- **修复建议**：在所有注册表路径操作前，验证路径不包含 `\..\`
- **预计工作量**：2-3 天

### 问题 8：DLL 劫持防护未实现
- **模块**：src/engine/process/process_inherit.cpp
- **问题描述**：未使用绝对路径加载 DLL，未验证 DLL 签名
- **影响**：安全漏洞，可能被 DLL 劫持攻击
- **修复建议**：使用绝对路径加载 DLL，验证 DLL 数字签名
- **预计工作量**：3-5 天

### 问题 9：权限提升检查未实现
- **模块**：src/engine/hook/hook_engine.cpp
- **问题描述**：未检查 Hook DLL 是否以管理员权限运行
- **影响**：安全风险，可能违反最小权限原则
- **修复建议**：在 Hook DLL 入口检查当前权限，拒绝以管理员权限运行
- **预计工作量**：1-2 天

### 问题 10：Windows 版本检查未实现
- **模块**：未实现
- **问题描述**：未检查 Windows 版本，可能无法在 Windows 10 22H2+ 上运行
- **影响**：兼容性问题，可能在某些 Windows 版本上崩溃
- **修复建议**：在程序入口调用 `IsWindows10OrGreater()` 和 `IsWindows1022H2OrGreater()`
- **预计工作量**：1 天

### 问题 11：基准应用验证未执行
- **模块**：tests/
- **问题描述**：未验证 Notepad++、7-Zip、Firefox Portable 在沙箱中的运行情况
- **影响**：兼容性问题，可能无法运行常见应用
- **修复建议**：创建测试脚本，自动化验证这 3 款应用
- **预计工作量**：3-5 天

### 问题 12：杀软兼容性测试未执行
- **模块**：未实现
- **问题描述**：未测试 Windows Defender、360、腾讯电脑管家、火绒的兼容性

## P1 问题（建议修复 - 影响用户体验）

### 问题 13：API 调用未全部检查返回值
- **模块**：所有模块
- **问题描述**：部分 API 调用（如 `wcsncpy_s`、`wcsncat_s`）未检查返回值
- **影响**：可能导致静默失败，难以调试
- **修复建议**：所有 API 调用检查返回值，使用 `assert()` 或错误日志
- **预计工作量**：2-3 天

### 问题 14：未使用 GetLastError() 获取错误码
- **模块**：所有模块
- **问题描述**：API 调用失败后，未使用 GetLastError() 获取详细错误码
- **影响**：难以诊断问题
- **修复建议**：在错误处理代码中调用 GetLastError()，记录到日志
- **预计工作量**：1-2 天

### 问题 15：VFS 缓存未实现
- **模块**：src/engine/vfs/vfs.cpp
- **问题描述**：未实现 VFS 缓存，每次文件操作都需要路径重定向计算
- **影响**：性能问题，文件操作延迟高
- **修复建议**：实现 LRU 缓存，缓存重定向后的路径
- **预计工作量**：3-5 天

### 问题 16：未实现事件驱动 Hook 覆盖检测
- **模块**：src/engine/hook/hook_engine.cpp
- **问题描述**：仅使用定时检测（每 5 秒），未使用事件驱动检测
- **影响**：Hook 被覆盖后最多需要 5 秒才能检测到
- **修复建议**：使用 `CreateToolhelp32Snapshot` 或 `PsSetCreateProcessNotifyRoutine` 实现事件驱动检测
- **预计工作量**：3-5 天

### 问题 17：未实现指数退避
- **模块**：src/engine/hook/hook_engine.cpp
- **问题描述**：Hook 重装重试间隔固定为 5 秒，未使用指数退避
- **影响**：频繁重试可能影响性能
- **修复建议**：实现指数退避（5s → 10s → 20s → 40s → 60s 封顶）
- **预计工作量**：1 天
