# Week 2 注册表重定向测试报告 (V3)

## 1. 测试概述

**测试时间**: 2026-05-23  
**测试人员**: AI Agent (Week 2)  
**测试环境**: Windows 10/11, Hook Engine (修改后)  
**测试程序**: `tests\engine\test_vreg.cpp`  

### 1.1 测试目的

验证注册表重定向功能是否正常工作：
1. 创建注册表键时，是否重定向到虚拟 hive
2. 设置注册表值时，是否写入虚拟 hive
3. 读取注册表值时，是否优先从虚拟 hive 读取（CoW 逻辑）
4. 删除注册表键时，是否仅删除虚拟 hive 中的键
5. 若虚拟 hive 不存在该键，是否从真实注册表读取（CoW）

### 1.2 测试范围

| 测试项 | 描述 | 优先级 |
|--------|------|--------|
| V3-1 | 创建注册表键重定向 | P0 |
| V3-2 | 设置注册表值重定向 | P0 |
| V3-3 | 读取注册表值（来自虚拟 hive） | P0 |
| V3-4 | 删除注册表键（仅删除虚拟 hive） | P0 |
| V3-5 | CoW 读取（虚拟 hive 不存在时透传） | P0 |

## 4. 代码修改说明

### 4.1 修复限制 1：注册表重定向为透传模式

**修改文件**: `src\engine\hook\vreg_engine.cpp`

**修改内容**:
1. 添加 JSON 助手函数（`JsonHelper` 命名空间）
2. 添加虚拟注册表操作函数：
   - `VRegCreateKey()` - 在虚拟 hive 中创建键
   - `VRegSetValue()` - 在虚拟 hive 中设置值
   - `VRegQueryValue()` - 从虚拟 hive 中查询值（CoW 逻辑）
3. 修改 Hook 函数：
   - `HookedNtCreateKey()` - 调用 `VRegCreateKey()`
   - `HookedNtSetValueKey()` - 调用 `VRegSetValue()`
   - `HookedNtOpenKey()` - 记录键打开操作
   - `HookedNtQueryValueKey()` - 调用 `VRegQueryValue()`（CoW）

**当前状态**:
- ✅ 代码已修改
- ⚠️ POC 阶段：仅记录操作到日志文件（`.log`）
- ⚠️ V2 阶段：需要实现完整的 JSON 解析和虚拟 hive 操作
