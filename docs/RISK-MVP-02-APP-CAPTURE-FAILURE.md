# 技术风险详述 - R-MVP-02（应用捕获失败率高）

## 风险描述

应用捕获工具（AppCapture）在捕获复杂应用时，可能因以下原因失败：
- 应用安装程序使用自定义安装逻辑（非标准 MSI）
- 应用运行时动态下载组件（如 Chrome 自动更新）
- 应用修改系统文件（如替换系统 DLL）
- 应用使用内核驱动（如杀毒软件、虚拟机）
- 应用安装过程需要重启（如驱动安装）

## 影响范围

- **影响用户**：需要使用非标准应用便携化的用户
- **影响功能**：应用捕获、.vapp 包生成
- **影响指标**：捕获成功率、.vapp 包质量

## 概率评估

- **技术难度**：高（应用安装逻辑千差万别）
- **类似项目参考**：Cameyo 捕获成功率约 70-80%（复杂应用需要手动调整）
- **概率**：**中**（40-60% 概率在 MVP 阶段捕获成功率 < 90%）

## 缓释措施（按优先级排序）

### P0（必须实现，MVP Phase 1 完成）

**措施 1：优先捕获 10 款常用应用**
- 选择兼容性好的应用（Notepad++, 7-Zip, VLC 等）
- 建立捕获模板库（每款应用的标准捕获流程）
- 预期捕获成功率：≥ 90%

**措施 2：实现快照对比增强**
- 不仅对比文件系统，还对比注册表、服务、驱动
- 使用多时间点快照（安装前、安装中、安装后、首次运行后）
- 预期捕获完整性提升 30%

### P1（建议实现，MVP Phase 2 完成）

**措施 3：实现手动调整界面**
- 用户可手动添加/删除捕获的文件/注册表项
- 提供"高级模式"（显示所有捕获项，用户可勾选）
- 预期捕获成功率提升 15%

**措施 4：建立兼容性规则库**
- 记录每款应用的特殊处理方式（如"应用 X 需要排除 Y 文件"）
- 社区贡献规则（GitHub 仓库）
- 预期长期捕获成功率提升至 95%

### P2（可选实现，MVP Phase 3 完成）

**措施 5：实现"录制模式"**
- 用户执行应用安装过程，系统录制所有操作
- 回放录制内容到新机器（类似 Dockerfile）
- 预期捕获成功率提升至 98%

## 应急预案（若缓释措施失效）

1. **降级方案**：提供"手动捕获模式"（用户自己指定需要捕获的文件/注册表项）
2. **回滚方案**：若某款应用无法捕获，标记为"不兼容"，在应用商店中隐藏
3. **用户沟通**：在文档中明确说明兼容性限制（如：不支持内核驱动类应用）

- 捕获成功率（目标：≥ 90%）
- 捕获时间（目标：< 5 分钟/应用）
- .vapp 包安装成功率（目标：100%）

若任意指标连续 3 次测试不达标，触发应急预案。
