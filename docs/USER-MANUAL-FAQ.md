## Q1：Launchpad 和 ThinApp 有什么区别？

**A**：

| 对比项 | AI ThinApp Portable Launchpad | VMware ThinApp |
|--------|-------------------------------|----------------|
| 价格 | 免费开源 | 商业软件（收费） |
| 协议 | GPL v3（完全开源） | 闭源商业软件 |
| 沙箱粒度 | 每个软件独立沙箱 | 每应用独立可执行文件 |
| 便携化方式 | 实时 Hook 重定向（运行时隔离） | 打包为单文件（静态封装） |
| 管理界面 | Launchpad 统一面板管理 | 无统一管理界面 |
| 软件商店 | ✅ 社区 .vapp 包分发 | ❌ 不支持 |
| 目标用户 | 个人用户 | 企业 IT 管理员 |
| 跨位置便携 | ✅ 移动应用目录后直接可用 | ✅ 单文件可直接分发 |

**简单说**：VMware ThinApp 是企业级应用打包工具（商业软件），而 AI ThinApp Portable Launchpad 是个人用户的免费替代方案，使用实时 Hook 技术而非静态封装，并且提供了统一的管理界面（Launchpad）。

## Q2：为什么我的应用无法启动？

**A**：请按以下顺序排查：

**第一步：基础检查**

| 检查项 | 操作 |
|--------|------|
| Launchpad 权限 | 确认以管理员身份运行 Launchpad（右键 → 以管理员身份运行） |
| 杀软白名单 | 确认 AI ThinApp 相关文件已在杀软白名单中 |
| 应用是否正常 | 尝试在资源管理器中直接运行该应用，确认应用本身没有问题 |

**第二步：切换模式测试**

1. 右键点击应用卡片 → "启动设置"
2. 切换为"真实模式"（不使用沙箱）
3. 尝试启动

- 如果真实模式下可以启动 → 问题出在沙箱引擎，请查看下方"沙箱模式相关问题"
- 如果两种模式都无法启动 → 应用本身可能有问题

**第三步：沙箱模式相关问题**

| 可能原因 | 解决方法 |
|----------|----------|
| Hook 注入失败 | 检查 Hook 引擎日志（设置 → Hook 设置 → 启用详细日志） |
| 沙箱数据损坏 | 清除沙箱数据后重试（应用详情页 → 清除沙箱） |
| 应用使用反作弊系统 | 反作弊系统可能与 Hook 冲突，只能以真实模式运行 |
| 应用以管理员权限运行 | 权限冲突，尝试关闭应用的管理员模式 |

**第四步：报告问题**

如果以上都无法解决，请到 [GitHub Issues](https://github.com/anmingwei/ai-thinApp-portable-lauchpad-platform/issues) 报告，附上：
- 应用名称和版本
- Windows 版本
- Hook 引擎日志（`%LOCALAPPDATA%\AI ThinApp\Logs\hook_engine.log`）
- 复现步骤

## Q3：为什么应用启动变慢了？

**A**：沙箱引擎会对文件/注册表操作进行拦截和重定向，会有一定性能开销。目标是将损耗控制在 **< 5%**。

**优化建议**：

| 操作 | 说明 |
|------|------|
| 关闭详细日志 | Hook 引擎的详细日志会显著影响性能，排查问题后请关闭 |
| 关闭不需要的重定向 | 如果应用不需要文件重定向，可以在应用设置中关闭"文件写入重定向" |
| 使用真实模式 | 对性能敏感的应用，可以切换到"真实模式"（不使用沙箱） |
| 关注更新 | 我们持续优化沙箱引擎性能，新版本会有改善 |

**常见性能开销来源**：

| 操作 | 目标增幅 |
|------|----------|
| 文件读取（沙箱检查） | < 5% |
| 文件写入（重定向） | < 10% |
| 注册表操作 | < 5% |
| 进程创建（Hook DLL 注入） | < 15%（仅创建时有损耗） |

> **注意**：MVP 阶段性能优化仍在进行中，部分场景的开销可能高于目标值。

## Q4：如何分享便携化应用给朋友？

**A**：有两种方式：

### 方式一：导出为 .vapp 包（推荐）

1. 在应用详情页，点击"导出为 .vapp"
2. 选择保存路径
3. 将 .vapp 文件发送给朋友（如通过微信、QQ、网盘）
4. 朋友在 Launchpad 中点击"+ 添加应用" → "从 .vapp 文件安装"
5. 选择 .vapp 文件，点击"安装"

**.vapp 包包含**：
- 应用文件
- 注册表数据
- 沙箱配置
- 元数据（应用名称、版本等）

### 方式二：复制整个应用目录

1. 在应用详情页，点击"打开目录"
2. 将整个应用文件夹复制给朋友（如通过 U 盘或网盘）
3. 朋友将文件夹放到电脑上
4. 在 Launchpad 中点击"+ 添加应用" → "浏览本地应用"
5. 选择应用的可执行文件

> **注意**：方式二需要朋友也安装 AI ThinApp Portable Launchpad。

## Q5：为什么杀毒软件报毒？

**A**：这是因为本产品使用 **API Hook 技术**，与一些病毒的行为模式类似（拦截系统 API 调用），因此可能被杀毒软件误报。

**安全措施**：

| 措施 | 说明 |
|------|------|
| 完全开源 | 所有代码在 [GitHub](https://github.com/anmingwei/ai-thinApp-portable-lauchpad-platform) 公开，任何人都可以审计 |
| 代码签名 | 正在申请 EV 代码签名证书，对可执行文件进行数字签名 |
| 白名单申请 | 正在与杀软厂商合作，将本项目加入白名单 |

**解决方法**：

| 步骤 | 操作 |
|------|------|
| 1 | 将 `C:\Program Files\AI ThinApp\` 添加到杀软白名单 |
| 2 | 将 `ai-thinapp.exe` 和 `hook_engine.dll` 单独添加到信任列表 |
| 3 | 如果被沙箱化的第三方应用也被误报，将该应用也加入白名单 |
| 4 | 报告误报到 [GitHub Issues](https://github.com/anmingwei/ai-thinApp-portable-lauchpad-platform/issues)，我们会联系杀软厂商 |

> **提示**：如果你不确定是否安全，可以查看项目源码（开源项目，代码完全透明），或请信任的技术人员帮忙检查。

## Q6：如何彻底删除沙箱？

**A**：

**方法一：通过 Launchpad 删除**（推荐）

1. 在"我的应用"列表中，右键点击应用卡片
2. 选择"删除应用"
3. 在确认对话框中，选择"删除应用和沙箱数据"
4. 点击"确认"

**方法二：仅清除沙箱数据（保留应用文件）**

1. 在应用详情页，点击"清除沙箱"
2. 确认对话框中显示将被清除的数据列表
3. 点击"确认清除"

清除后，应用文件保留，但所有配置和数据回到初始状态。

**方法三：手动删除**

1. 在应用详情页，点击"打开目录"
2. 删除 `VFS` 文件夹（即沙箱数据）
3. 或直接删除整个应用文件夹

> **注意**：删除沙箱数据不可恢复，建议先导出备份。

**A**：

| Windows 版本 | 支持状态 | 说明 |

## Q8：可以便携化游戏吗？

**A**：**部分可以，部分不可以**。

**可以便携化的游戏**：

- 不使用反作弊系统的游戏（如独立游戏、单机游戏）
- 普通 PC 游戏（如 Minecraft、Stardew Valley）
- 支持绿色版的游戏

**不可以便携化的游戏**：

- 使用反作弊系统（如 Easy Anti-Cheat、BattlEye）的游戏
- 反作弊系统通常会直接使用 Syscall（绕过 ntdll），用户态 Hook 无法拦截
- 如：Valorant、Fortnite、PUBG 等在线竞技游戏

**建议**：

- 便携化游戏前，请先尝试捕获和测试
- 如果游戏使用反作弊系统，建议以真实模式运行（不使用沙箱）
- 关注后续版本更新，我们计划在内核驱动层面增强兼容性（V2）
