# 杀软兼容性测试计划 (Antivirus Test Plan)

> 本文档定义 AI ThinApp Portable Launchpad Platform 项目的杀软兼容性测试方案。
> 杀软误报是本项目的高风险项（R-PROJ-02，等级：极高），必须认真验证。

---

## 1. 测试目标

- 验证 Hook DLL（hook_engine.dll）在主流杀软下的误报情况
- 验证测试程序（test_hook.exe, test_process.exe）的误报情况
- 记录误报类型（恶意/可疑/无需操作）
- 制定缓解措施（代码签名、开源透明、申请白名单）

### 3.1 测试用例详细说明

#### TC-01: Hook DLL 静态扫描

**测试步骤**：
1. 访问 https://www.virustotal.com/
2. 上传 `hook_engine.dll`
3. 等待扫描完成（通常 1-2 分钟）
4. 记录检测结果：
   - 检测率（多少引擎报毒）
   - 报毒类型（恶意/可疑/无需操作）
   - 报毒引擎名称

**通过标准**：
- 检测率 ≤ 5%（即 ≤ 3/60 引擎）
- 无主流杀软报毒（Defender、火绒、360、卡巴斯基、诺顿）

**失败处理**：
- 若检测率 > 5%，分析报毒原因（特征码误判？行为可疑？）
- 考虑代码混淆、加壳、或重写可疑代码段

#### TC-03: Hook DLL 动态加载

**测试步骤**：
1. 启动测试程序（test_hook.exe）
2. 程序自动加载 hook_engine.dll
3. 观察杀软反应（实时防护日志）
4. 若杀软报毒，记录：
   - 报毒时机（加载时？运行时？）
   - 报毒类型（恶意/可疑/无需操作）
   - 杀软建议操作（隔离/允许/阻止）

**通过标准**：
- 杀软不报毒，或仅首次提示用户放行
- 用户选择"允许"后，程序可正常运行

**失败处理**：
- 若杀软阻止加载，尝试：
  1. 代码签名后重新测试
  2. 申请杀软白名单
  3. 提示用户添加信任

## 5. 缓解措施

| 措施 | 优先级 | 成本 | 效果 | 实施难度 |
|------|--------|------|------|----------|
| 代码签名（OV） | P0 | ~$200/年 | 减少误报 50-80% | 低 |
| 代码签名（EV） | P1 | ~$400/年 | 减少误报 80-95% | 中（需硬件 Token） |
| 开源透明（GitHub） | P0 | 免费 | 增加信任度，减少误报 20-30% | 低 |
| 申请杀软白名单 | P1 | 免费（需审核） | 减少误报 90-100%（针对特定杀软） | 高（审核周期长） |
| 提示用户关闭杀软 | P0 | 免费 | 临时方案，用户体验差 | 低 |
| 使用合法证书签名（EV） | P1 | ~$400/年 | 微软 SmartScreen 不拦截 | 中 |

### 5.1 缓解措施详细说明

#### 代码签名（OV/EV）

**OV（Organization Validation）证书**：
- 费用：~$200/年
- 验证：企业身份（营业执照、电话验证）
- 效果：减少误报 50-80%，SmartScreen 可能仍提示
- 适用：POC 阶段、MVP 阶段

**EV（Extended Validation）证书**：
- 费用：~$400/年
- 验证：企业身份 + 法律文件 + 硬件 Token
- 效果：减少误报 80-95%，SmartScreen 不拦截
- 适用：正式发布阶段

**申请流程**：
1. 选择 CA 机构（DigiCert / Sectigo / GlobalSign）
2. 提交申请材料：
   - 企业营业执照
   - 法定代表人身份证
   - 电话验证（CA 机构回拨）
3. 等待审核（3-5 工作日）
4. 收到证书（OV：邮件下载；EV：硬件 Token 邮寄）
5. 使用 SignTool 对 DLL/EXE 进行签名

#### 开源透明（GitHub）

**策略**：
- 公开所有源码（GitHub 仓库）
- 提供构建脚本（任何人可复现构建）
- 提供校验和（SHA-256）验证发布文件未被篡改

**效果**：
- 增加信任度（用户可审计代码）
- 减少误报 20-30%（杀软厂商会参考 GitHub 仓库）
- 社区可帮助诊断误报问题

**实施步骤**：
1. 创建 GitHub 仓库（public）
2. 上传所有源码（包括 Hook 引擎、测试程序）
3. 编写 README.md（项目介绍、构建步骤）
4. 提供构建脚本（build.bat / build.sh）
5. 发布 Release（附带 SHA-256 校验和）

#### 申请杀软白名单

**Defender Application Control**：
- 申请地址：https://www.microsoft.com/en-us/wdsi/filesubmission
- 提交文件：hook_engine.dll（已签名）
- 审核周期：3-5 工作日
- 通过率：高（若代码签名 + 开源透明）

**火绒白名单申请**：
- 申请地址：https://www.huorong.cn/feedback.html
- 提交文件：hook_engine.dll（已签名）
- 审核周期：1-2 工作日
- 通过率：中等（需提供详细解释）

**360 白名单申请**：
- 申请地址：https://cert.360.cn/
- 提交文件：hook_engine.dll（已签名）
- 审核周期：3-7 工作日

## 附录：VirusTotal 使用指南

### A.1 上传文件

1. 访问 https://www.virustotal.com/
2. 点击 "Choose File"
3. 选择要扫描的文件（hook_engine.dll）
4. 等待上传和扫描完成

### A.2 解读报告

- **Detection ratio**：检测率（例如 2/61，表示 61 个引擎中有 2 个报毒）
- **Details**：每个引擎的检测结果
  - **Undetected**：未检测到威胁
  - **Malicious**：恶意（确认病毒）
  - **Suspicious**：可疑（可能病毒）
  - **Harmless**：无害
  - **Timeout**：扫描超时

### A.3 误报申诉

如果确认是误报，可以：
1. 联系报毒引擎的厂商（例如 Microsoft、火绒）
2. 提交误报申诉（提供文件 + 解释）
3. 等待厂商更新病毒库（通常 1-3 天）
