# 安全披露政策 (Security Policy)

## 报告安全漏洞

如果您发现 AI ThinApp Portable Launchpad Platform 存在安全漏洞，请通过以下方式向我们报告：

### 联系方式

- **电子邮件**: security@example.com （请替换为实际邮箱）
- **GitHub Security Advisories**: [点击此处报告](https://github.com/用户名/ai-thinapp-portable-launchpad/security/advisories/new)

### 报告内容

请包含以下信息：

- 漏洞类型和位置
- 重现步骤
- 潜在影响
- 建议的修复方案（可选）
- 您的联系方式（用于后续沟通）

## 响应时间

- **确认收到**: 48 小时内
- **初步评估**: 7 天内
- **修复发布**: 根据严重程度，30-90 天内

## 披露流程

1. **报告**: 您通过电子邮件或 GitHub Security Advisories 报告漏洞
2. **确认**: 我们在 48 小时内确认收到报告
3. **评估**: 我们进行初步评估，确定严重程度和影响范围
4. **修复**: 我们开发并测试修复方案
5. **披露**: 我们与您协调披露时间，通常会在修复发布后公开披露
6. **致谢**: 我们会在安全公告中致谢（除非您要求匿名）

## 安全最佳实践

使用本项目时，请遵循以下安全最佳实践：

1. **仅从官方渠道下载**: 只从 GitHub Releases 或官方构建下载二进制文件
2. **验证签名**: 检查代码签名（如果可用）
3. **最小权限原则**: 不要以管理员权限运行，除非必要
4. **沙箱测试**: 在隔离环境中测试未知应用程序
5. **及时更新**: 保持更新到最新版本

## 依赖项安全

我们定期通过以下方式检查依赖项的安全漏洞：

- **Dependabot**: 自动检查依赖项漏洞
- **定期审计**: 手动审计第三方代码

如果您发现依赖项存在安全漏洞，请通过上述方式报告。
